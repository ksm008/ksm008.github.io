처음에는 불린 배열을 만들어서 페어면 불린 값을 바꾸고 마지막에 반복문으로 찾는 방법을 생각했다.

```java
import java.io.*;  
  
public class Main {  
    static BufferedReader br = new BufferedReader(new InputStreamReader(System.in));  
    static BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));  
    static StringBuilder sb = new StringBuilder();  
  
    public static void main(String[] args) throws IOException {  
        boolean[] isNotPair = new boolean[10];  
        int N;  
  
        for (int i = 0; i < 5; i++) {  
            N = Integer.parseInt(br.readLine());  
  
            if (isNotPair[N] == false) isNotPair[N] = true;  
            else isNotPair[N] = false;  
        }  
  
        for (int i = 0; i < 10; i++) {  
            if (isNotPair[i] == true){  
                sb.append(i);  
                break;  
            }  
        }  
  
        bw.write(sb.toString());  
        bw.flush();  
        bw.close();  
        br.close();  
    }  
}
```

하지만 배열 대신 XOR 연산을 사용하면 마지막 반복문은 필요없게 된다. XOR 연산은 각 숫자가 같으면 결과가 0이 되고, 만약 숫자와 0을 연산하게 되면 그 숫자가 그대로 튀어나온다. 따라서...

```java
import java.io.*;  
  
public class Main {  
    static BufferedReader br = new BufferedReader(new InputStreamReader(System.in));  
    static BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));  
    static StringBuilder sb = new StringBuilder();  
  
    public static void main(String[] args) throws IOException {  
        int result = 0;  
        for (int i = 0; i < 5; i++) {  
            int N = Integer.parseInt(br.readLine());  
  
            result ^= N;  
        }  
  
        sb.append(result);  
        bw.write(sb.toString());  
        bw.flush();  
        bw.close();  
        br.close();  
    }  
}
```

그냥 XOR 계산식 하나면 해결되는 문제기도 하다.
