## # @Builder

빌더 어노테이션은 롬복에서 제공하는 기능 중 하나이다. 자바의 디자인 패턴 중 빌터 패턴을 자동으로 만들어준다. (GoF의 디자인 패턴 중 생성 패턴에 해당함)

### 개념
객체를 만들 때 생성자의 단점을 보완하기 위해 사용하는 패턴. 

#### 생성자가 불편한 이유
만약 엔티티에 필드가 10개가 넘어간다면 가독성이 나빠진다.

```java
// 문제점 1: 너무 복잡함
new Post("제목", "내용", null, null, "사진", 0, true, "일반", "김철수", null);

// 문제점 2: 가독성이 나쁨 (순서가 헷갈림)
// 이 'true'가 '비밀글 여부'인지 '공지 여부'인지 알 수가 없음.
```

#### 빌더 패턴의 장점
빌더 패턴은 new Post(...) 를 직접 호출하지 않는다. 대신 중간 단계의 빌더 객체를 만들어서 필드를 하나씩 체인(chain) 형태로 설정한 후 마지막에 build() 메소드를 호출하여 객체를 완성한다.

 - **가독성이 좋아짐 :** 어떤 필드에 어떤 값을 넣는지 명확하게 드러나 코드를 읽기 쉽고 순서도 상관없다.

```java
Post.builder()
    .title("제목입니다")       // 어떤 필드인지 명확함
    .content("내용입니다.")   // 어떤 필드인지 명확함
    // .password("1234")      // 선택적인 필드는 생략 가능
    .build();
```

- **안정성이 높고 유연함** - 생성자는 인수의 순서를 지켜야 하지만 빌더는 순서가 상관없다. 또한 원하는 필드만 골라서 객체를 만들 수 있으므로 생성자를 여러 개 생성하지 않아도 된다.
