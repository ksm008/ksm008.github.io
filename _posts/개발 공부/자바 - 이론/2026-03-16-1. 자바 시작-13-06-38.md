## 1.1 컴퓨터와 프로그래밍
### 컴퓨터와 소프트웨어
대충 하드웨어 소프트웨어 프로그램 이야기.

### 프로그래밍 언어
- 컴퓨터가 실행할 프로그램을 작성하는 언어
- CPU는 0과 1밖에 이해하지 못해서 이진수로만 구성된 언어를 **기계어**라고 한다.
- 사람이 기계어를 작성하는 것은 어렵기 때문에 기계어의 각 명령을 ADD, SUB, MOVE 등 표현하기 쉬운 **니모닉 기호**(mnemonic symbol)로 일대응 대응시킨 **어셈블리어**가 만들어졌다.
- 근데 그 어셈블리어도 사람이 다루기 힘들고, 다양한 프로그램이나 자료를 표현하기에는 한계가 있어서 사람이 이해하기 쉽고 복잡한 알고리즘이나 프로그램, 자료를 효율적으로 표현할 수 있는 고급 언어가 생겨나게 됨.
- 고급 언어는 초기에 절차 지향을 거쳐 지금의 객체 지향 언어로 진화함.
- C++, C#, 자바는 대표적인 객체 지향 언어이다. 자바는 C/C++ 언어 구조를 차용한다.

### 프로그램 컴파일과 실행
- 프로그래밍은 컴퓨터 언어로 프로그램 소스 파일을 작성하는 것으로 시작함. 
- 컴퓨터는 C/C++, 자바와 같은 고급 언어로 작성된 프로그램은 직접 처리할 수 없기 때문에, 컴퓨터가 실행할 수 있는 기계어 코드로 변환하는 **컴파일** 과정이 필요함.
- 컴파일을 전담하는 소프트웨어를 **컴파일러**라고 한다. 
- 각 언어마다 전용 컴파일러가 있다. 자바는 소스 프로그램의 확장자가 **.java**이고, 자바 전용 컴파일러에 의해 **.class** 파일로 컴파일되며, 클래스 파일은 자바 가상 기계에 의해 실행된다. 

## 1.2 자바의 출현과 WORA
- 자바는 애초에 PC나 인터넷 환경에 사용될 목적으로 설계된 것이 아님.
- 1991년 선마이크로시스템즈의 **제임스 고슬링**이라는 사람이 그린 프로젝트라는 가전제품 소프트웨어 개발 프로젝트를 시작했었음
- 기존 프로그래밍 언어로 작성된 프로그램들은 플랫폼 간에 호환성이 없어 PC의 플랫폼에 따라 소스를 다시 컴파일하거나 아예 프로그램을 재작성해야하는 단점이 있었다. 
- 동시에 가전제품의 메모리는 매우 적은 양의 메모리를 사용해야 했다.
- 그래서 선마이크로시스템즈는 플랫폼 독립적 + 메모리 사용량이 적은 새로운 언어, 실행 체계를 개발했고 처음에는 그 이름을 **oak**라고 지었다.
- 처음에는 별로 알려지지 않았다. 하지만 인터넷과 웹의 발전으로 급속도로 퍼지기 시작함.
- 왜냐면 웹의 특징이 웹 콘텐츠가 클라이언트의 다양한 플랫폼상 웹 브라우저에서 실행되도록 하는 것인데 이 플랫폼 독립성이 딱 맞았기 때문.
- 그래서 그걸 발전시켜서 인터넷 환경에 적합하도록 만들고 1995년에 **자바**라는 이름으로 새롭게 발표함.
- 당시 가장 많이 사용되던 넷스케이프에 자바 기술을 적용하면서 급격히 퍼짐. 
- 오라클에서 2000년에 선마이크로시스템즈를 인수하면서 지금은 **오라클**에서 자바를 제공하고 있음.

### WORA(Write Once Run Anywhere)
- C/C++ 등 자바 이전의 프로그래밍 언어들의 컴파일러는 기계어 코드를 변환할 때 프로그램이 실행될 대상 컴퓨터의 CPU와 운영체제에 종속적인 코드로 변환한다.
- 그래서 다른 CPU, 운영체제에서 실행하려면 다시 컴파일을 하거나 코드 자체를 수정해야했다.
- 하지만 자바는 개발 단계부터 플랫폼 독립적으로 설계가 되었다. 운영체제나 하드웨어에 상관없기 자바 가상 기계인 **JVM**이 있다면 어디서나 자바로 작성된 프로그램에 동일한 실행 환경을 제공하여 동일한 실행 결과를 기대할 수 있다.
- 자바 프로그램은 어디서든 한 번 작성하면 다시 컴파일하거나 수정하지 않고 어떤 하드웨어에서나 운영체제에서도 실행시킬 수 있다. 이걸 **WORA(Write Once Run Anywhere)** 라고 한다.

### 자바 가상 기계와 바이트 코드
- 플랫폼 독립성을 가능하게 한 것은 자바 가상 기계와 바이트 코드.
- 자바 소스 코드를 컴파일하면 클래스 파일이 되고, 그 클래스 파일에는 바이트 코드가 들어있다.

#### 바이트 코드
- 자바 가상 기계에서만 실행되는 기계어.
- 어떤 CPU와는 관계없는 바이너리 코드. 
- 자바 컴파일러는 소스 프로그램을 컴파일 하여 바이트 코드로 된 클래스 파일을 생성한다.
- 이 클래스 파일은 CPU에 의해 직접 실행되지 않고 자바 가상 기계가 인터프리터 방식으로 실행시킨다.
- 오라클에서 배포하는 **JDK**에는 자바 클래스 파일을 디어셈블하여 바이트 코드를 볼 수 있는 도구인 **javap**를 제공한다.

## 1.3 개발 도구와 자바 플랫폼
### JDK와 JRE
- JDK는 자바 개발자를 위한 상업용 소프트웨어. 자바 컴파일러 등의 개발 도구와 JRE로 구성되어있다.
- JRE는 자바 응용 프로그램이 실행될 때 필요한 소프트웨어들로 자바 API(이미 컴파일된 다양한 클래스 라이브러리)와 자바 가상 기계를 포함한다.
- 자바 프로그램을 개발하지 않고 실행시키기만 하는 일반 사용자의 경우 JRE만 필요하다.
- 개인의 경우 비상업적인 경우 무료로 사용이 가능하지만 상업적 목적이나 대학과 같은 기관의 경우 JDK 사용은 유료임. 근데 그냥 교육용으로 쓸려면 OpenJDK를 쓰면 된다.
