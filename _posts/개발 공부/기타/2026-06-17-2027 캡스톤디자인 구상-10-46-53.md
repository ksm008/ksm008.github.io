## **1. 프로젝트 기획 배경 및 목적**
최근 Figma, 웹 기반 Photoshop 등 고성능 연산이 필요한 웹 서비스의 구현을 위해 웹어셈블리(WASM) 기술 도입이 급증하고 있다. 그러나 해커들은 WASM의 빠른 연산 속도와 기계어로 컴파일된 바이너리 형태라는 특성을 악용하여 악성 코드를 은닉하는 기법을 사용하기 시작했다. 기존 텍스트 기반의 웹 방화벽(WAF)이나 상용 백신으로는 이를 분석하고 탐지하는 데 한계가 명확하다.

보안을 이유로 방화벽 단에서 WASM 실행을 원천 차단할 경우, 정상적인 상용 업무 도구까지 사용이 불가능해지는 서비스 가용성 침해 문제가 발생한다. 따라서 정지된 파일의 외형만 검사하는 '정적 분석'의 한계를 극복하고, 격리된 가상 환경에서 파일을 직접 실행하며 이상 행위를 실시간으로 추적하는 '행위 기반 동적 분석 샌드박스'의 개발이 필수적이다.

## **2. 시스템 아키텍처 및 기술 스택**
시스템은 데이터 분석, 서버 제어, 사용자 인터페이스를 담당하는 3개의 주요 계층으로 구성된다.
- **핵심 분석 엔진 (Python)**
    - `wasmtime-py` 라이브러리를 활용하여 WASM 파일을 안전하게 실행할 수 있는 격리된 샌드박스 환경을 구축한다.
    - `psutil`을 통해 실행 중인 프로세스의 CPU 및 메모리 점유율을 측정하고, 허가되지 않은 메모리 접근이나 위험한 운영체제 시스템 호출(WASI)을 실시간으로 감시한다.
    - 심층적인 난독화 해독이 필요한 경우, 고성능 LLM API(Claude 3.7 Sonnet 등)로 코드를 전송하여 악성 로직 여부를 교차 검증한다.
- **백엔드 제어 서버 (Java & Spring Boot)**
    - 사용자의 분석 요청을 수신하고 파이썬 분석 엔진의 실행을 통제하는 중앙 관리 서버 역할을 수행한다.
    - 다수의 분석 요청이 몰릴 경우 대기열(Queue)을 생성하여 순차적으로 처리하며, 최종 분석 결과와 로그를 데이터베이스에 저장한다.
- **사용자 대시보드 (React & Vite)**
    - 사용자가 의심되는 WASM 파일을 업로드할 수 있는 직관적인 웹 인터페이스를 제공한다.
    - 샌드박스에서 수집된 파일의 CPU/메모리 변화량을 실시간 시계열 그래프로 시각화하고, 최종 산출된 위험도 점수를 화면에 출력한다.
- **인프라 환경 (Docker)**
    - 분석 엔진이 악성 파일을 직접 다룬다는 점을 고려하여, 파이썬 실행 환경을 Docker 컨테이너로 캡슐화한다. 이를 통해 Host OS와의 완벽한 격리(이중 방어막)를 구현하여 시스템 보안성을 강화한다.

## **3. 기대 효과**
본 시스템을 도입함으로써 정상적인 웹 서비스와 악성 공격을 정교하게 식별할 수 있다. 기업 및 교육 기관은 필수적인 웹 기반 업무 도구의 가용성을 유지하면서, 사용자 PC 자원을 탈취하는 크립토재킹(암호화폐 채굴) 및 브라우저 메모리 공격을 효과적으로 방어할 수 있다. 또한, 특정 악성코드의 시그니처에 의존하지 않고 실제 발생하는 '행위'를 기반으로 탐지하므로, 기존에 알려지지 않은 신종 해킹 기법(Zero-day)에 대해서도 유연하고 선제적인 대응이 가능하다.

## **4. 한계점 및 극복 방안**
- **한계점:** 동적 분석은 파일을 가상 환경에서 직접 실행해야 하므로 시스템 자원(CPU, RAM) 소모가 크다. 악의적으로 설계된 무한 루프 코드가 유입될 경우 서버 과부하 및 다운 위험이 존재하며, LLM API를 활용한 난독화 분석 시 추가적인 시간 지연과 API 호출 비용이 발생할 수 있다.
- **극복 방안:** 소프트웨어적 안전장치를 도입하여 물리적 한계를 보완한다. 분석 파일의 최대 실행 시간을 제한하는 **'타임아웃(Timeout)'** 로직을 적용하여 과부하를 원천 차단하고, 서버가 안정적으로 처리할 수 있는 가용 자원 범위 내에서만 작업을 수행하도록 **'비동기 큐(Queue) 대기열'** 시스템을 구현하여 서비스의 신뢰성을 확보한다.
