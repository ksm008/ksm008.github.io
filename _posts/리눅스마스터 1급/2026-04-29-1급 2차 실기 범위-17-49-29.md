### Snort
- Snort는 탐지 룰(Rule)을 이용하여 네트워크 트래픽을 분석하고 침입을 탐지할 수 있다. 
- **Rule의 구조 :** 헤더와 룰 옵션으로 구분됨.
- **헤더**
	- action : 동작 방식. 주로 alert만 나옴. 
		- **alert : 경고 발생**
		- log : 경고 발생 없이 로그에만 기록
		- pass : 통과
		- activate : 경고 발생 및 이와 연결된 dynamic 룰 활성화
		- dynamic : activate에 의해 활성화 되었을때만 패킷을 로그에 기록
		- drop : 패킷을 차단하고 로그에 남김. 응답 메시지 없음.
		- reject : 패킷 차단 + 로그 남기고 연결 거부 응답 메시지를 보냄.
		- sdrop : 패킷 차단만, 로그 없음.
	- protocol : 점검 대상 프로토콜
	- source IP/Port : 소스 IP 및 포트
	- direction : 패킷의 흐름 방향
	- destination IP/Port : 목표 IP 및 포트
- **룰 옵션**
	- General
	- Payload
	- non-payload
	- post-detection
	- 총 4가지로 분류 가능
	- 주로 이중 Payload 탐지 룰 옵션을 사용함.
- **주요 형태**
	- 룰 옵션의 주요 항목을 비워놓고 작성하는 형식임.
	- **msg :** 메세지를 지정함
	- **sid :** 설정한 룰의 식별자를 지정하며 사용자가 작성한 룰은 1,000,000 이상으로 함.
	- **content :** 페이로드 내부에서 검색할 문자열을 지정함.
	- depth : 검사할 바이트를 지정
	- offset : 검사를 시작할 위치를 지정
	- distance : 앞에서 설정한 룰의 결과에서 지정한 숫자(바이트)만큼 떨어진 지점부터 검사함.
	- within : 시작점부터 검사하되 지정한 바이트 내에서만 검사.
	- **nocase :** 문자열 검사시 대소문자 구별 안함.
	- sameip :  소스와 목표 IP 주소가 동일한 것을 검사함.

#### 예제 문제 
외부에서 내부 대역의 80 포트로 이동하는 패킷 중 "passwd" 문자열이 포함된 패킷이 발견되면 "passwd detected" 라는 경고 메시지를 발생시킴. 단, passwd는 대소문자를 구분하지 않고 비교하여 포함 여부를 판단한다.

`( 1 ) tcp any any -> 192.168.10.0/24 80 (( 2 ):"passwd detected"; ( 3 ):"passwd"; ( 4 ); sid:1000001;)`

- 1 : alert
- 2 : msg
- 3 : content
- 4 : nocase

만약 나오면 대충 이런 느낌인 것 같다.

### iptables
- 리눅스의 방화벽이다.
- **테이블 종류**
	- **filter**
		- 네트워크 패킷을 대상으로 정해진 규칙을 수행하고 패킷 필터링, 방화벽 기능을 수행한다.
		- **INPUT**
			- 호스트를 목적지로 들어오는 패킷을 필터링함. 인바운드.
		- **OUTPUT**
			- 호스트를 출발지로 나가는 패킷을 필터링하는데 사용. 아웃바운드.
		- **FORWARD**
			- 라우터로 사용되는 호스트를 통과하는 패킷이다. 호스트 컴퓨터가 목적지가 아닌 패킷을 관리함. 내가 라우터 기능을 할 때 쓰는건가?
	- **nat**
		- 네트워크 어드레스 트랜스레이션. 네트워크 주소 즉 IP 주소 및 포트를 변환하고 관리한다. 하나의 공인 IP를 여러 호스트가 사용하고자 할 때 주로 사용.
		- **PREROUTING**
			- 패킷의 도착지 주소를 변경함.
		- **POSTROUTING**
			- 출발지 주소를 변경한다. masquerade라고도 함. 웬 가면무도회
		- **OUTPUT**
			- 호스트에서 밖으로 흐르는 패킷의 도착지 주소를 변경함.
		- **INPUT**
			- 호스트 밖에서 안으로 흐르는 패킷의 주소를 변경.
	- mangle
	- raw
	- security
	- 나머지 3개는 출제가 잘 안되는 모양.
- **사용 방법**
	- `iptables [-t 테이블 이름] [액션] [체인 이름] [매치 규칙] [-j 타깃]` 형식을 이용.
	- filter 테이블은 기본값임. -t filter를 쓰지 않아도 자동으로 필터로 적용이 된다.
	- **체인을 대상으로 한 액션의 항목**
		- -N : 새로운 정책 체인을 만듬. new.
		- -X : 인풋, 아웃풋, 포워드를 제외한 비어있는 정책 체인을 제거함. execute의 약자라고 생각하기
		- -L : 현재 정책 체인 목록을 표시. 리스트.
		- -F : 지정된 체인에 설정된 모든 정책을 삭제. 플러시.
		- -C : 패킷을 테스트함. 체크의 약자인듯
		- -P : 체인의 기본 정책을 설정. 폴리시.
		- -Z : 체인 내의 모든 규칙들의 패킷, 바이트 카운터를 0으로. 제로의 약자
	- **체인 내부를 대상으로 한 주요 액션 항목**
		- -A : 어펜드. 새로운 정책을 가장 마지막에 추가 등록.
		- `-I [체인] [라인번호]` : 지정한 라인 번호에 추가. 인서트.
		- 그 외에 딜리트, 리플레이스 약자인 -D, -R도 위와 동일함.
	- **주요 매치 규칙 항목**
		- -s : 출발지 ip, 도메인 설정
		- -d : 목적지 ip, 도메인 설정
		- ! : 이기적 미친놈들아 느낌표는 부정이지 뭔 뒤 규칙은 제외야 ㅅㅂㅋㅋㅋㅋ 
		- -p : tcp, udp, icmp등 프로토콜 설정
		- -i : 입력 네트워크 인터페이스 설정
		- -o : 출력 네트워크 인터페이스 설정
		- --sport : 출발지(소스) 포트 설정, ':'을 사용. 목적지는 당연히 dport겠죠?
		- --tcp-flags : SYN, ACK등 TCP flag를 지정함
		- --syn : TCP flag로 SYN만 가진 것을 지정
		- --icmp-type : ICMP 타입을 지정함. echo-request(8), recho-reply(0), destination-unreachable(3)등이 있음.
		- -m : 특정 모듈, 규칙과 매치함.
		- --state : 연결 상태와 매치함
		- --string : 응용 계층의 데이터 바이트 순서와 매치함
	- **기타 부가 옵션 항목**
		- -n : 출력 정보를 숫자로 표시
		- -v : 부가 정보를 표시
		- --line-numbers : 정책 앞에 번호를 붙여서 표시. -L 과만 쓰인다.
	- **-j 타깃에 사용할 수 있는 항목**
		- ACCEPT : 패킷 허용
		- REJECT : 패킷 거부. TCP일 경우 reset, UDP일 경우 Port unreachable 응답 메시지 전송.
		- DROP : 패킷 거부하고 버림. 응답 메시지 없음.
		- LOG : 패킷을 syslog에 전달하여 기록. 경로는 **/var/log/message**
		- RETURN : 호출 체인 내에서 패킷을 계속 진행함.
	- 우왕 뒤지게 많아요!
- `iptables -A INPUT -s 192.168.10.7 -d localhost -j DROP`
- P는 기본적인 정책을 다루는 옵션이므로 -j의 옵션만 쓴다.
	- `iptables -P OUTPUT ACCEPT`
	- 다른 규칙항목은 쓸 수 없다.
- `iptables -A INPUT -s 192.168.10.7 -d localhost -j DROP`
- `iptables -A INPUT -s 192.168.10.7 -p icmp -j REJECT`
- `iptables -A OUTPUT -o eth0 -d www.youngjin-test.com -p tcp --dport 80 -j DROP`
- **설정 저장**
	- `iptables-save > firewall.sh`
	- iptables-save는 저장하는 명령어.
- **설정 적용**
	- `iptables-restore < firewall.sh`
	- iptables-restore는 불러오는 명령어.
- **설정 자동 적용**
	- `service iptables save`
	- 정책을 **/etc/sysconfig/iptables**에 저장할 수 있다. 
	- 서비스 시작 시 자동으로 읽어들여 적용됨.
	- 근데 사실상 록키 리눅스 8에서는 작동하지 않음. 뭐 어쩌라는거임? 이럴거면 왜 알려주냐
- **NAT 구성**
	- -t nat로 설정하는거.
	- SNAT이냐 DNAT이냐가 중요함. 
		- SNAT : Source NAT. 소스 IP 주소를 변경함.
		- DNAT : Destination NAT. 목적지 IP 주소를 변경함.
	- SNAT은 **POSTROUTING**, DNAT은 **PREROUTING**.
	- -j 옵션 뒤에 SNAT, DNAT을 선택하여 써주면 된다. 
	- 책에 보면 --to MASQUERADE라고 되어있는 부분이 있는데... 애초에 MASQUERADE는 주소가 아니라서 -j MASQUERADE로 써야 한다. 
		- `iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE`

```bash
...
[root@localhost ~]# iptables -t nat -A POSTROUTING -o eth0 -j SNAT --to MASQUERADE
iptables v1.8.5 (nf_tables): Bad IP address "MASQUERADE"
```

- **이어서...**
	- 그리고 또 SNAT의 경우 **--to-source**로, DNAT의 경우 **--to-destination**으로 목적지를 지정해줘도 되지만... iptables가 그냥 자동으로 변환해주기 때문에 그냥 **--to**만 써도 무방.
	- SNAT일때는 -o, DNAT일때는 -i로 인터페이스 지정.


> **리눅스에서 NAT을 사용하려면...**<br>
> - **/proc/sys/net/ipv4/ip_forward의 값이 1로 설정이 되어있어야 한다.**

### firewalld
- 네트워크 연결의 신뢰도에 따라 영역을 구분하여 사용하므로 방화벽 규칙을 효율적으로 관리할 수 있음.
- **영역 확인 명령어**
	- `ls -l /usr/lib/firewalld/zones/`
- **주요 명령어**
	- 이 책에서 firewalld는 세세하게 옵션을 알려주지 않음. 출제가 많이 된 적이 없는건지
	- `firewall-cmd --state` : 상태를 확인
	- `firewall-cmd --reload` : 설정 다시 로드. 영구 옵션 없는 규칙은 사라짐.
	- `firewall-cmd --get-zones` : 존 목록을 출력
	- `firewall-cmd --get-default-zone` : 기본 존 출력
	- `firewall-cmd --get-active-zones` : 활성화된 존 출력
	- `firewall-cmd --add-service=http --add-service=https` : HTTP, HTTPS 서비스 추가
	- `firewall-cmd -add-port=21/tcp` : TCP/21 포트를 추가함
	- `firewall-cmd --zone=public --list-all` : 퍼블릭 존의 사용 가능한 서비스, 포트 목록을 출력한다
	- `firewall-cmd --remove-port=21/tcp` : --zones 옵션이 없으면 기본 존을 대상으로 함. TCP/21 포트 삭제.
	- `firewall-cmd -add-port=8080-8090/tcp` : 포트 범위를 지정하여 추가
	- `firewall-cmd --permanent --add-service=ftp` : FTP를 영구적으로 추가
	- `firewall-cmd --set-default-zone=block` : 기본 존을 block으로 변경. 외부 접근을 다 무시함. reject여서 응답을 보냄.
	- `firewall-cmd --add-forward-port=port=80:proto=tcp:toport=8080` : tcp/80으로 들어오는 패킷을 8080포트로 포워딩함.
	- `firewall-cmd --permanent -add-imcp-block=echo-request` : ICMP Echo-Request에 응답하지 않음.
