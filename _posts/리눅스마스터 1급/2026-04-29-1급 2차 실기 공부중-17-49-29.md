### tar - 백업
- **-c** : 아카이브 생성
- **-f** : 아카이브 명 지정
- **-v** : 상세 정보 출력
- -p : 접근권한 유지.
	- ex) `tar -cvf [아카이브 파일명] [대상]`
- -x : 아카이브 파일 추출
	- ex) `tar -xvf archive.tar`
- -z : .gz 형식으로 압축
- -j : .bz2 형식으로 압축.
- -J : .xz 형식으로 압축.
	- ex) `tar -cvjfp archive.tar.bz2 /path/to/dir`
	- ex) `tar -cvzfp archive.tar.gz /path/to/dir`
	- ex) `tar -cvJfp archive.tar.xz /path/to/dir`
- -g : 증분백업. 변경된 파일 및 디렉터리 정보를 스냅샷으로 보관한다.
- -t : 아카이브 내용 확인
### cpio
- cpio는 하나의 디렉터리에서 다른 디렉터리로 파일을 복사하기 위한 목적으로 개발됨.
- 원래는 표준 입력으로 입력을 받음.
- -o : out 모드. 아카이브 생성
- -i : in 모드. 아카이브 추출.
- -v : 처리 내용 출력
- -F : 아카이브할 파일 이름을 지정
- -p : 패스모드. 복사할 때 사용
- -t : 아카이브 내용 확인
- -H : 아카이브 포맷 지정. newc, bin, odc...등등
- -d : 디렉터리를 생성. 난 또 따로 디렉토리명을 지정해줘야 하는줄 알았네...
- **예시**
	- `find . -name "*.txt" | cpio -ov > backup.cpio`
	- `cpio -iv < backup.cpio`
	- `cpio -ovF archive`
	- `<` 가 먼저, `>` 가 이후. 입력 후 출력이다.
	- 백업을 할 때는 >, 백업에서 불러올 때는 <를 쓴다.
	- `cpio -ov < newText.txt > archive.tar = cpio -ovF archive.tar < newText.txt`
		- 서로 같다.

### dump
- 파티션 단위 백업.
- **주요 옵션**
	- 0 ~ 9 : 레벨을 지정. 0단계는 전체, 1단계부터는 부분 백업.
	- -f : 백업할 장치나 파일을 지정.
	- -u : 덤프 완료 후 /etc/dumpdates 라는 파일에 백업 정보를 기록.
	- `dump -0uf [백업을 저장할 파일이나 디렉터리] [절대경로]`
- **예시**
	- `dump -0u -f backup.dump /dev/hda7`
	- `dump -0f /dev/nst0 /home`
	- `dump -0uf` 이런식으로 붙여써도 되는듯... 단, f가 마지막에 와야 한다.

### dd
- 지정한 장치를 지정한 형식으로 변환함.
- 파티션이나 디스크 단위로 백업.
- **주요 옵션**
	- if : 표준 입력 대신 지정한 파일이나 장치에서 읽음.
	- of : 표준 출력 대신 파일이나 장치로 씀
	- bs : 블록을 읽거나 쓸 때 크기를 지정함. 단위는 바이트.
- **예시**
	- `dd if=/dev/sda1 of=/dev/sdb1 bs=1k`
		- /dev/sda1 장치를 /dev/sdb1 장치로 백업하고 블록 크기는 1kb로 설정.
	- `dd if=/dev/sda of=~/disk.img`
		- /dev/sda 장치를 홈 디렉터리의 disk.img 파일로 백업.

### restore
- dump 명령으로 생성한 백업 파일을 복원할 때 사용함.
- **주요 옵션**
	- -i : 대화 모드로 복원.
	- -f : 복원할 파일을 지정.
	- -r : 전체 복원.
- **예시**
	- `restore -f backup.dump` : backup.dump 복원

### rsync
- 네트워크로 연결된 리모트 컴퓨터의 파일들을 동기화하는 도구.
- `rsync [옵션] [원본] [목적지]`
- **주요 옵션**
	- -a : 여러파일을 하나로 묶는 아카이브 모드로 동작한다. -rlptgoD를 하나로 묶은것과 같다.
	- -v : 자세히 출력
	- -z : 압축을 전송.
- **예시**
	- `rsync -avz root@192.168.5.13:/home /` : 원격지의 홈 디렉터리를 로컬로 복사
	- `rsync -avz /data/backup /backup_dir` : 로컬 디렉터리 동기화 백업.

### HTTP, Apache <<< 필수
- **응답 코드 :**
	- 대충 다음과 같다
		- 100번대 : 조건부 응답
		- 200번대 : 성공
		- 300번대 : 리다이렉션
		- 400번대 : 요청오류
		- 500번대 : 서버오류
	- 100번대
		- 101 : 컨티뉴
		- 101 : 프로토콜 변경
	- 200번대
		- 200 : 전송 성공
		- 201 : 생성됨
		- 202 : 허용됨
		- 203 : Non-authoriative information
		- 204 : No content
		- 205 : Reset Content
		- 206 : Partial Content
	- 300번대
		- 300 : Multiple Choices
		- 301 : Moved Permanently
		- 302 : Moved Temporary
		- 303 : See Other
		- 304 : Not Modified
		- 305 : Use Proxy
	- 400번대
		- 400 : Bad Request (클라 문법 오류)
		- 401 : Unauthorized (권한이 없어 인증 실패)
		- 402 : Payment Required (결재 필요 요청)
		- **403 :** Forbidden (리소스에 대한 권한이 없어 인가 실패)
		- **404 :** Not Found (존재하지 않음)
		- 405 : Method Not Allowed (해당 방식으로 요청 불가)
		- 408 : 타임 아웃
	- 500번대 
		- 500 : 서버 내부 오류
- **/usr/local/apache/conf :**  httpd.conf가 담겨있는 디렉터리. 다른 추가 설정 파일들은 **extra** 폴더에 들어있음. 
- **/usr/local/apache/bin/apachectl**
	- restart : 재시작
	- -M : 로드된 모듈 출력
	- configtest : httpd.conf의 문법 오류 점검
	- graceful : 연결을 끊지 않고 설정을 재적용
- **/usr/local/apache/bin/httpd
	- -t : 문법 오류를 점검
	- -f : 특정 환경설정 파일을 지정하여 아파치 데몬을 실행
	- -S : 현재 설정된 가상 호스트 목록을 출력
	- -M : 로드된 모듈 목록을 출력
- **주요 환경설정 파일**
	- **httpd.conf** : 기본 설정 파일. 이 아래부터 파일은 앞에 전부 `httpd-`가 붙는다. 귀찮아서 생략한거임
	- mpm : MPM 설정 파일
	- multilang-errordoc : 에러처리에 대한 다국어 문서 처리
	- autoindex : 디렉터리 목록을 표시하는 방법을 설정하는 파일
	- **userdir** : 사용자의 홈 디렉터리를 설정하는 파일
	- info : 웹 서버의 실행 상태 모니터링을 위한 설정 파일
	- **vhosts** : 버추얼 호스트를 설정하는 파일
	- dav : WebDAV를 설정하는 파일. webDAV는 웹 서버에서 파일 목록의 조회, 수정, 삭제 등을 할 수 있는 프로토콜.
	- default : 웹 서버의 기본 설정값을 설정하는 파일
	- language : 언어를 설정하는 파일
	- manual : 메뉴얼 문서를 지원하기 위한 설정 파일
	- ssl :  SSL 지원을 위한 설정 파일
- **httpd.conf 설정 항목들**
	- **ServerRoot :** 주요 파일들이 저장된 초상위 디렉터리를 절대 경로로 지정.
	- **ServerAdmin :** 서버 관리자 메일 주소를 설정.
	- **ServerName :** 서버의 
	- **Listen :** 웹 서버가 사용할 포트 지정
	- **LoadModule** : 모듈 로딩.
		- `LoadModule userdir_module modules/mod_userdir.so` 같은 식으로 쓴다. 모든 모듈은 **/usr/local/apache/bin/apachectl restart :** 안에 mod_모듈이름.so 형식으로 저장되어 있음.
	- **DocumentRoot :**  웹 문서가 저장되는 기본 디렉터리 경로를 저장. 기본은 **/var/www/html**
		- `DocumentRoot "/var/www/html"` - 얘는 `""`를 붙여야 한다.
	- **DirectoryIndex :** 처음 웹을 방문할 때 열릴 파일 목록을 정의. 여러개 나열 가능. 기본 인덱스.
		- `DirectoryIndex index.html index.htm index.php
	- **Include :** 별도의 설정 파일을 포함.
		- ex) `Include conf/extra/httpd-userdir.conf`
	- ErrorLog : 에러 로그 파일 위치 지정
	- LogLevel : 그 로그를 기록하는 수준을 결정. 
	- DefaultType : 기본 MIME 타입을 지정. 
	- AddType : 파일 확장자에 대한 MIME을 등록.
	- **AccessFileName :** 디렉터리별 접근 제어할 때 사용. .htaccess가 기본.
	- **ErrorDocument :** 에러시 일반 오류 메시지를 출력하거나, 특정 페이지 또는 외부 사이트로 리다이렉트 되도록 설정할 수 있다.
		- ex) `ErrorDocument 500 "this server is potato"`
		- ex) `ErrorDocument 404 /missingContent.html`
		- ex) `ErrorDocument 402 http://www.errorsite.com/noPayment.html`
	- 그 외 User, Group, AddDefaultCharset. TypesConfig, Redirect 등이 있지만 최근 5년간 출제되지 않음.
- **httpd-userdir.conf 설정 항목**
	- **UserDir**
		- 사용자 웹 디렉터리 지정 지시어.
		- `UserDir disabled` : 사용하지 않음. 단 `UserDir disabled 계정명` 형태로 특정 사용자만 제외도 가능...
		- `UserDir [폴더명]` : 웹 페이지를 저장할 기본 경로는 public_html이다. 해당 디렉터리에 대한 접근 권한을 허가해둬야 한다.
- **httpd-vhosts.conf 설정 항목**
	- 버츄얼 호스트를 이용하여 하나의 IP 주소로 여러 도메인을 설정할 수 있다. 
	- `<VirtualHost *:80>` : IP 주소, 포트 번호와 함께 가상호스트 추가
	- ServerName : 도메인 이름 지정
	- ServerAlias : 별칭 추가할 때 사용
	- `httpd -S` 명령으로 설정 상태 확인 가능
- **htpasswd**
	- 아파치 웹 서버의 특정 페이지에 사용자 인증을 설정할 수 있다.
	- `htpasswd -c /etc/passwd master` : 마스터 사용자 생성과 계정 파일로 /etc/password 파일을 사용한다.
	- **httpd.conf 설정**
		- 대상 디렉터리의 AllowOverride 항목에 **AuthConfig**를 지정한다.
	- **.htaccess :** 디렉터리별 접근 제어 파일
	- **.htaccess 파일 주요 설정 항목**
		- **AuthType** : 인증 방식을 지정.
		- **AuthName** : 인증 창에 표시될 이름을 설정.
		- **AuthUserFile** : 사용자 계정 파일을 지정
		- **Require**  : 접속을 허가할 사용자를 지정. AuthUserFile을 사용... `Require vaild-user`를 하면 사용자 계정 파일에 등록된 모든 사용자를 허가하고, 그게 아니면 `Require 사용자1, 사용자2` 이런식으로 쓴다.
	- **htpasswd의 주요 옵션**
		- -c : 사용자 계정 파일을 생성.
		- -D : 사용자를 계정 파일에서 제거
		- -s : SHA 암호화로 사용자 계정 파일을 암호화함.

### SAMBA
- 삼바 서버는 **/etc/samba/smb.conf**를 환경설정 파일로 이용한다.
- **smb.conf의 주요 구성 항목**
	- 글로벌 섹션, 쉐어 데피니션 영역으로 구분되며 `[]` 기호를 통해 세부 섹션을 구성한다.
	- **글로벌 섹션의 주요 설정 항목**
		- 전체적으로 다 한번씩 나온적은 있으니 다 외워두는게 좋다.
		- workgroup : 공유 그룹명. 윈도우의 작업 그룹과 동일.
		- server string : 서버에 대한 설명
		- netbios name : 이름을 이용하여 접속할 때 사용하는 명칭
		- interfaces : 사용할 네트워크 페이스 설정. 
		- hosts allow : 삼바 서버에 접근할 수 있는 호스트를 지정
		- log file : 삼바 서버의 로그 파일을 지정.
		- max log size : 로그 파일 최대 크기
		- **securtiy :** 클라이언트가 삼바 서버에 접근할 때 인증 레벨을 부여하는 보안 옵션 설정. 기본값은 user.
			- user : 삼바 서버에 접속할 시 OS에 이미 로그인한 사용자명으로 패스워드를 확인함.
			- share : 인증없이 삼바 서버에 접근.
			- server : 윈도우 서버와 같은 다른 삼바 서버에 사용자명과 페스워드를 전달하여 확인, Deprecated됨.
			- domain : 윈도우 서버의 도메인 컨트롤러에 사용자명, 패스워드를 전달하여 확인. 
	- **쉐어 데피니션의  주요 설정 옵션**
		- `[디렉터리 이름]` : 대괄호 사이에 공유 폴더 이름을 지정함. **<< 많이 나옴.**
			- ex) `[homes]`
		- **comment :** 공유 폴더에 대한 설명을 기술
		- **path :** 공유 디렉터리의 절대 경로를 지정
		- read only = yes : 읽기만 가능하도록 설정하기
		- writeable = yes : 쓰기 가능하도록 설정
		- **write list = `[사용자명]` :** 쓰기 가능한 사용자를 지정, @를 사용하여 그룹을 지정할 수 있음.
		- **vaild users = `[사용자명]` :** 접근 가능한 사용자를 지정. 별도 지정 없으면 모든 사용자가 접근 가능함
		- public = no : 개인 사용자만 사용할 수 있도록 설정
		- browseable : 이용 가능한 공유 리스트에 표지되지 않도록.
		- 그 외에 printable = yes, create mask = 값, follow symlinks = no 등이 있다.
- **smbpasswd : 삼바 계정과 패스워드를 등록**
	- -a : 삼가 사용자 계정 추가
	- -x : 삭제
	- -d : 비활성화
	- -e : 활성화
	- -n : 패스워드를 제거함. 단, smb.conf 설정 항목으로 `null passwords = yes`를 추가해야함.
- **smbclient : 삼바의 공유 디렉터리 정보 확인, 서버 접속**
	- -L : 삼바 서버의 공유 디렉터리 정보를 표시
	- -M : Ctrl + d와 함께 사용하여 메세지를 전송.
	- -U : 사용자 이름을 지정.
	- -p `[TCP 포트]` : 서버의 TCP 포트를 지정.
- **그 외 기타 명령어**
	- smbstatus : 현재 접속 정보를 확인
	- testparm : 삼바의 설정 정보를 확인하는 명령어.
	  nmblookup : 삼바 서버의 NetBIOS 이름으로 IP 주소를 조회할 수 있는 명령어
	- smbcontrol : 삼바 데몬에 메세지를 보내어 제어할 수 있음.
- **서버 접속**
	- `smbclient //서버아이피/공유명 -U 사용자명%패스워드`


### NFS
- rpcbind, nfs-utils 패키지를 사용한다.
- **/etc/exports** 파일을 이용한다.
- `[공유 디렉터리][접속 허가 호스트] (옵션)` 형식으로 이용하여 공유할 목록을 하나씩 지정.
	- ex) `/home/youngjin-nfs 172.30.1.0/24(rw,no_root_squash)`
	- **주요 옵션**
		- no_root_squash : 클라이언트의 root 사용자가 서버에서도 root 권한을 유지함
		- root_squash : root 사용자도 일반 사용자 권한으로 매핑함. 
		- all_squash : 모든 사용자의 권한을 일반 사용자 권한 (nobody)으로 매핑시킴.
		- no_all_squash : 위의 반대. 일반 사용자 권한으로 매핑하지 않음.
		- ro, rw : 각각 읽기 전용, 읽기 및 쓰기 옵션 설정
		- no_subtree_check : 하위 디렉터리 검사를 금지
		- async : 비동기처리
		- sync : 동기적으로 처리
		- secure : 포트가 1024 이하의 요청만 허가
		- anonuid : 특정 계정의 권한을 할당
- **기타 명령어**
	- **rpcinfo :** RPC 관련 정보를 표시함
	- **exportfs :** NFS 서버의 익스포트된 디렉터리 정보, 즉 공유 목록을 관리함
	- **showmount :** 마운트 관련 정보를 표시하는 명령어
	- **nfsstat :** NFS 관련 상태정보를 표시하는 명령어.


### VSFTP
- FTP 패키지. 
- **주요 파일, 디렉터리**
	- **/etc/vsftpd/vsftpd.conf :** 주 설정파일
	- **/etc/pam.d/vsftpd :** PAM 설정 파일
	- **/etc/vsftpd/ftpusers :** FTP 서비스에 **접근 할 수 없는** 사용자 계정을 지정함. 접근할수없는!!
	- **/etc/vsftpd/user_list :** 특정 사용자만 잡속을 허용하기 위한 파일 경로. 하지만 설정법에 따라서 어떻게 작동하는지가 다름. 주 설정 파일에서`userlist_enable=YES`로 설정되어 있어야 작동함. 기본값은 설정되어있음.  `userlist_deny=YES`라면 user_list에 등록된 사용자는 접근할 수 없음. 블랙리스트가 되어버림. `userlist_deny=NO`로 설정하면 반대로 화이트리스트가 된다.
- **/etc/vsftpd/vsftpd.conf의 주요 설정 항목**
	- anonymous_enable=YES : 익명 사용자 접속 허가
	- local_enable=YES : 로컬 사용자 FTP 접속 허용   
	- chroot_local_user=YES : 로컬 사용자를 홈 디렉터리에 가둠   
	- allow_writeable_chroot=YES : chroot 환경에서 쓰기 권한 허용


### sendmail
- 주 경로는 /etc/mail/
- **주요 설정 파일**
	- **/etc/mail/sendmail.cf** 
		- 기본 설정 파일
	- **/etc/mail/sendmail.mc** 
		- sendmail의 설정을 편리하게 할 수 있는 보조 파일.
		- `m4 /etc/mail/sendmail.mc > /etc/mail/sendmail.cf` : m4 유틸리티를 사용하여 sendmail.cf를 생성한다.
		- sendmail-cf 패키지가 필요.
	- **/etc/aliases**
		- 메일의 별칭, 혹은 특정 계정으로 수신한 이메일을 다른 계정으로 전달하도록 설정. 보통 여러 사람에게 전달할 때 사용함.
		- `[수신 계정] : [전달 계정]` 형식을 따른다. 
		- `:include:[파일 이름]`으로 사용자 이름이 저장된 파일을 설정할 수 있다.
		- sendmail이 참조하는 파일은 /etc/aliases.db이므로 /etc/aliases를 수정한 후 newaliases나 sendmail -bi 명령으로 적용한다.
	- **/etc/mail/access** 
		- 메일 서버에 접속하는 호스트의 접근을 제어하는 설정 파일. 스팸 메일 방지 등에 사용 가능.
		- `[정책 대상][정책]`의 형식을 사용. 
		- 정책 대상은 도메인명, 아이피, 메일 주소를 사용. 
		- 정책은 아래와 같다.
			- RELAY (릴레이 허용)
			- DISCARD (거부)
			- 거부 후 메시지 전송 (REJECT)
			- DNS 조회 실패 시에도 허용 (OK)
		- 위 정책을 지정할 수 있다.
		- `makemap hash /etc/mail/access < /etc/mail/access` 같은 명령으로 **/etc/mail/access.db**에 적용함.
	- **/etc/mail/virtusertable**
		- 가상의 메일 계정으로 들어오는 메일을 특정 계정으로 전달하는 정보를 설정.
		- `makemap hash /etc/mail/virtusertable < /etc/mail/virtusertable` 와 같은 명령으로 **/etc/mail/virtusertable/db**에 적용한다.
	- **/etc/mail/local-host-names**
		- sendmail에서 수신할 메일의 도메인과 호스트, 즉 메일 수신지를 설정하며 sendmail을 다시 시작하여 적용한다.
	- **~/.forward**
		- 사용자 개인이 수신한 메일을 다른 메일로 포워딩할 때 설정하는 파일. 원하는 메일 주소를 한 줄씩 추가함
- **/etc/mail/sendmail.cf** 파일 건드리는건 웬만해서 안나오는 것 같다. 전날에나 좀 볼 예정.

### **DNS**
- DNS 관련 패키지 이름은 **bind**, 데몬 이름은 **named**
- **주요 파일, 디렉터리**
	- **/etc/named.conf**
		- 존 파일, 리버스 존 파일을 비롯한 환경 설정하는 파일.
		- 존 파일은 도메인 이름, IP 주소, 관련 리소스 간 매핑을 포함하고 리소스 레코드로 구성됨.
		- 리버스 존 파일을 이용하여 IP 주소에 대한 도메인 정보 조회를 제공할 수 있음.
	- **/var/named**
		- 루트 도메인 서버의 정보를 담은 named.ca, 사용자가 설정한 zone 파일 등을 저장하는 디렉터리.
	- **bind-chroot** 패키지를 사용하면 보안을 향상히킬 수 있고, 위 경로가 달라진다.
		- **/etc/named.conf -> /var/named/chroot/etc/named.conf**
		- **/var/named -> /var/named/chroot/var/named**
	- **/etc/named.rfc1912.zones :** 존 설정 분리 파일. 
		- `include "/etc/named.rtf1912.zones";` 구문을 named.conf 마지막에 넣어 다른 존 정보를 추가한다.
- **/etc/named.conf 파일 설정**
	- option, acl, logging, zone 등의 구문으로 나뉨.
	- **option 구문 설정 항목**
		- directory : zone 파일 저장 디렉터리. 필수 항목.
		- dump-file : 정보 갱신 시 저장 파일로 사용할 덤프 파일의 파일명을 지정.
		- statictics-file : 통계 정보를 저장할 파일명
		- memstatictics-file : 메모리 통계 정보를 저장할 파일명
		- secroots-file : secroot 파일명
		- recursing-file :  recursing query 정보를 저장하는 파일명
		- **forward :**  forwarders 옵션과 함께 사용하며 only, first 값을 가진다.
			- only : 도메인 주소에 대한 질의를 다른 서버에게 넘김
			- first : 다른 서버에서 응답이 업을 경우 자신이 응답하도록 설정
		- **forwarders**
			- forward를 처리할 서버를 지정함. 세미콜론으로 여러개의 서버를 구분하여 설정 가능.
			- ex) `forwarders { 8.8.8.8; 8.8.4.4; };`
		- **allow-query**
			- DNS 질의를 허용할 대상을 지정.
			- ex) `allow-query {192.168.12/24;};`
		- **allow-transfer**
			- 존 파일 내용을 복사할 대상을 제한.
			- ex) `allow-transfer {172.3.1.0/24;};`
		- **allow-recursion :** 재귀 질의를 허용할 대상 지정. `allow-recursion {any;};`
		- datasize : 캐싱에 사용하는 메모리 크기 제한. `datasize 128M`
		- recursion : 하위 도메인 검색 허용 여부를 지정. `recursion yes` 같이 지정하면 캐싱 네임 서버의 역할로 동작하여 DNS 질의를 수행한다.
	- **acl 구문 설정 항목**
		- 여러 호스트들을 하나의 이름으로 지정하여 option 구문의 allow-query, allow-transfer 등에 사용할 수 있도록 함. option 구문 이전에 설정해야한다.
		- ex) `acl "ihd" {192.168.2.24; 192.168.4/24;};`
	- **zone 구문 설정 항목**
		- 형식은 `zone [도메인명] IN { type [타입옵션]; file [존 파일명];};`이다.
		- zone에 . 이 들어간다면 루트 도메인을 의미한다.
		- 리버스 존을 쓸 때는 `zone "3.10.210.in-addr.arpa" IN` 처럼 가장 뒷자리 삭제 + 뒤집은 다음 뒤에 in-addr.arpa를 붙여야 한다. 만약 아이피가 210.10.3.5라면 위의 결과가 된다.
		- type
			- hint, master, slave 3가지로 나뉨. 
			- hint : 루트 도메인을 지정
			- master : 1차 네임 서버를 지정
			- slave : 2차 네임 서버를 지정
		- file : 존 파일명.

- **zone 파일 설정**
	- 일반 Zone 파일은 .zone, 리버스 존은 .rev이다.
- **zone 파일의 구성 항목**
	- 세미콜론으로 주석을 달 수 있다.
	- **도메인은 언제나 맨 뒤에 점을 붙인다.** 
	- **$TTL :** 타임 투 라이브의 약자. 설정한 정보를 다른 곳에서 조회했을 경우 캐시에 보관할 시간을 지정. 기본 단위는 초. 분, 시간 단위로도 설정 가능. 존 파일의 첫 번째 라인에 위치함.
	- **SOA 레코드 :** 스타트 오브 어토리티의 약자. 도메인 관리자 메일, 일련번호 등 DNS의 핵심 정보를 지정. 
		- @ 기호는 /etc/named.conf에서 설정한 오리진 도메인을 의미함. 도메인을 직접 지정할 때에는 마지막에 반드시 . 을 붙여야 함.
	- **개별 도메인 (리소스 레코드)**
- **SOA 레코드의 주요 속성**
	- **네임 서버, 어드민 이메일 주소**
		- `@ IN SOA ns.ihd.or.kr. admin.ihd.or.kr.`
		- SOA임을 명시하고 네임 서버, 어드민 이메일 주소를 차례로 쓴다.
		- `kait@ihd.or.kr` 형식을 DNS 형식으로 작성한 것이 `kait.ihd.or.kr.` 이다.
		- 어드민 이메일은 이메일 주소 앞에 admin을 붙인다.
	- 이후에 오는 속성들은 차례대로 순서를 지켜야 한다.
		- 시리얼 번호 : Zone 파일 갱신시 증가하는 일련번호
		- 리프레시 넘버 : 업데이트 새로고침 
		- 재시도 주기
		- 만료 시간
		- 최소보관시간
- **개별 도메인의 주요 속성**
	- **주요 레코드 타입**
		- NS : 해당 존을 담당하는 네임 서버 지정
		- CNAME : 도메인 이름의 별칭
		- A : 도메인 이름에 해당하는 아이피
		- SOA : 존의 기본 관리 정보
		- MX : 메일 서버 지정.
		- PTR : IP 주소를 가지고 도메인 반환 (리버스 존)
- **DNS 관련 주요 명령어**
	- **named-checkzone :** zone 파일의 문법 오류 점검
		- ex) `named-checkzone ihd.or.kr /var/named/ihd.zone`
	- **named-checkconf :** /etc/named.conf 환경설정 파일의 문법적 오류 점검
	- **rndc status :** DNS 서버의 상태 확인
	- **host :** 도메인을 조회. dig 명령을 이용하면 더 자세한 정보 확인 가능

### Snort - 안나오는 추세인것 같음...
- Snort는 탐지 룰(Rule)을 이용하여 네트워크 트래픽을 분석하고 침입을 탐지할 수 있다. 
- **Rule의 구조 :** 헤더와 룰 옵션으로 구분됨.
- **헤더**
	- action : 동작 방식. 주로 alert만 나옴. 
		- **alert : 경고 발생**
		- log : 경고 발생 없이 로그에만 기록
		- pass : 통과
		- activate : 경고 발생 및 이와 연결된 dynamic 룰 활성화
		- dynamic : activate에 의해 활성화 되었을때만 패킷을 로그에 기록
		- drop : 패킷을 차단하고 로그에 남김. 응답 메시지 없음.
		- reject : 패킷 차단 + 로그 남기고 연결 거부 응답 메시지를 보냄.
		- sdrop : 패킷 차단만, 로그 없음.
	- protocol : 점검 대상 프로토콜
	- source IP/Port : 소스 IP 및 포트
	- direction : 패킷의 흐름 방향
	- destination IP/Port : 목표 IP 및 포트
- **룰 옵션**
	- General
	- Payload
	- non-payload
	- post-detection
	- 총 4가지로 분류 가능
	- 주로 이중 Payload 탐지 룰 옵션을 사용함.
- **주요 형태**
	- 룰 옵션의 주요 항목을 비워놓고 작성하는 형식임.
	- **msg :** 메세지를 지정함
	- **sid :** 설정한 룰의 식별자를 지정하며 사용자가 작성한 룰은 1,000,000 이상으로 함.
	- **content :** 페이로드 내부에서 검색할 문자열을 지정함.
	- depth : 검사할 바이트를 지정
	- offset : 검사를 시작할 위치를 지정
	- distance : 앞에서 설정한 룰의 결과에서 지정한 숫자(바이트)만큼 떨어진 지점부터 검사함.
	- within : 시작점부터 검사하되 지정한 바이트 내에서만 검사.
	- **nocase :** 문자열 검사시 대소문자 구별 안함.
	- sameip :  소스와 목표 IP 주소가 동일한 것을 검사함.

#### 예제 문제 
외부에서 내부 대역의 80 포트로 이동하는 패킷 중 "passwd" 문자열이 포함된 패킷이 발견되면 "passwd detected" 라는 경고 메시지를 발생시킴. 단, passwd는 대소문자를 구분하지 않고 비교하여 포함 여부를 판단한다.

`( 1 ) tcp any any -> 192.168.10.0/24 80 (( 2 ):"passwd detected"; ( 3 ):"passwd"; ( 4 ); sid:1000001;)`

- 1 : alert
- 2 : msg
- 3 : content
- 4 : nocase

### iptables
- 리눅스의 방화벽이다.
- **테이블 종류**
	- **filter**
		- 네트워크 패킷을 대상으로 정해진 규칙을 수행하고 패킷 필터링, 방화벽 기능을 수행한다.
		- **INPUT**
			- 호스트를 목적지로 들어오는 패킷을 필터링함. 인바운드.
		- **OUTPUT**
			- 호스트를 출발지로 나가는 패킷을 필터링하는데 사용. 아웃바운드.
		- **FORWARD**
			- 라우터로 사용되는 호스트를 통과하는 패킷이다. 호스트 컴퓨터가 목적지가 아닌 패킷을 관리함. 내가 라우터 기능을 할 때 쓰는건가?
	- **nat**
		- 네트워크 어드레스 트랜스레이션. 네트워크 주소 즉 IP 주소 및 포트를 변환하고 관리한다. 하나의 공인 IP를 여러 호스트가 사용하고자 할 때 주로 사용.
		- **PREROUTING**
			- 패킷의 도착지 주소를 변경함.
		- **POSTROUTING**
			- 출발지 주소를 변경한다. masquerade라고도 함. 웬 가면무도회
		- **OUTPUT**
			- 호스트에서 밖으로 흐르는 패킷의 도착지 주소를 변경함.
		- **INPUT**
			- 호스트 밖에서 안으로 흐르는 패킷의 주소를 변경.
	- mangle
	- raw
	- security
	- 나머지 3개는 출제가 잘 안되는 모양.
- **사용 방법**
	- `iptables [-t 테이블 이름] [액션] [체인 이름] [매치 규칙] [-j 타깃]` 형식을 이용.
	- filter 테이블은 기본값임. -t filter를 쓰지 않아도 자동으로 필터로 적용이 된다.
	- **체인을 대상으로 한 액션의 항목**
		- -N : 새로운 정책 체인을 만듬. new.
		- -X : 인풋, 아웃풋, 포워드를 제외한 비어있는 정책 체인을 제거함. execute의 약자라고 생각하기
		- -L : 현재 정책 체인 목록을 표시. 리스트.
		- -F : 지정된 체인에 설정된 모든 정책을 삭제. 플러시.
		- -C : 패킷을 테스트함. 체크의 약자인듯
		- -P : 체인의 기본 정책을 설정. 폴리시.
		- -Z : 체인 내의 모든 규칙들의 패킷, 바이트 카운터를 0으로. 제로의 약자
	- **체인 내부를 대상으로 한 주요 액션 항목**
		- -A : 어펜드. 새로운 정책을 가장 마지막에 추가 등록.
		- `-I [체인] [라인번호]` : 지정한 라인 번호에 추가. 인서트.
		- 그 외에 딜리트, 리플레이스 약자인 -D, -R도 위와 동일함.
	- **주요 매치 규칙 항목**
		- -s : 출발지 ip, 도메인 설정
		- -d : 목적지 ip, 도메인 설정
		- ! : 이기적 미친놈들아 느낌표는 부정이지 뭔 뒤 규칙은 제외야 ㅅㅂㅋㅋㅋㅋ 
		- -p : tcp, udp, icmp등 프로토콜 설정
		- -i : 입력 네트워크 인터페이스 설정
		- -o : 출력 네트워크 인터페이스 설정
		- --sport : 출발지(소스) 포트 설정, ':'을 사용. 목적지는 당연히 dport겠죠?
		- --tcp-flags : SYN, ACK등 TCP flag를 지정함
		- --syn : TCP flag로 SYN만 가진 것을 지정
		- --icmp-type : ICMP 타입을 지정함. echo-request(8), recho-reply(0), destination-unreachable(3)등이 있음.
		- -m : 특정 모듈, 규칙과 매치함.
		- --state : 연결 상태와 매치함
		- --string : 응용 계층의 데이터 바이트 순서와 매치함
	- **기타 부가 옵션 항목**
		- -n : 출력 정보를 숫자로 표시
		- -v : 부가 정보를 표시
		- --line-numbers : 정책 앞에 번호를 붙여서 표시. -L 과만 쓰인다.
	- **-j 타깃에 사용할 수 있는 항목**
		- ACCEPT : 패킷 허용
		- REJECT : 패킷 거부. TCP일 경우 reset, UDP일 경우 Port unreachable 응답 메시지 전송.
		- DROP : 패킷 거부하고 버림. 응답 메시지 없음.
		- LOG : 패킷을 syslog에 전달하여 기록. 경로는 **/var/log/message**
		- RETURN : 호출 체인 내에서 패킷을 계속 진행함.
	- 우왕 뒤지게 많아요!
- `iptables -A INPUT -s 192.168.10.7 -d localhost -j DROP`
- P는 기본적인 정책을 다루는 옵션이므로 -j의 옵션만 쓴다.
	- `iptables -P OUTPUT ACCEPT`
	- 다른 규칙항목은 쓸 수 없다.
- `iptables -A INPUT -s 192.168.10.7 -d localhost -j DROP`
- `iptables -A INPUT -s 192.168.10.7 -p icmp -j REJECT`
- `iptables -A OUTPUT -o eth0 -d www.youngjin-test.com -p tcp --dport 80 -j DROP`
- **설정 저장**
	- `iptables-save > firewall.sh`
	- iptables-save는 저장하는 명령어.
- **설정 적용**
	- `iptables-restore < firewall.sh`
	- iptables-restore는 불러오는 명령어.
- **설정 자동 적용**
	- `service iptables save`
	- 정책을 **/etc/sysconfig/iptables**에 저장할 수 있다. 
	- 서비스 시작 시 자동으로 읽어들여 적용됨.
	- 근데 사실상 록키 리눅스 8에서는 작동하지 않음. 뭐 어쩌라는거임? 이럴거면 왜 알려주냐
- **NAT 구성**
	- -t nat로 설정하는거.
	- SNAT이냐 DNAT이냐가 중요함. 
		- SNAT : Source NAT. 소스 IP 주소를 변경함.
		- DNAT : Destination NAT. 목적지 IP 주소를 변경함.
	- SNAT은 **POSTROUTING**, DNAT은 **PREROUTING**.
	- -j 옵션 뒤에 SNAT, DNAT을 선택하여 써주면 된다. 
	- 책에 보면 --to MASQUERADE라고 되어있는 부분이 있는데... 애초에 MASQUERADE는 주소가 아니라서 -j MASQUERADE로 써야 한다. 
		- `iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE`

```bash
...
[root@localhost ~]# iptables -t nat -A POSTROUTING -o eth0 -j SNAT --to MASQUERADE
iptables v1.8.5 (nf_tables): Bad IP address "MASQUERADE"
```

- **이어서...**
	- 그리고 또 SNAT의 경우 **--to-source**로, DNAT의 경우 **--to-destination**으로 목적지를 지정해줘도 되지만... iptables가 그냥 자동으로 변환해주기 때문에 그냥 **--to**만 써도 무방.
	- SNAT일때는 -o, DNAT일때는 -i로 인터페이스 지정.
	- 근데 대부분 SNAT은 --to를 쓰고 DNAT일떄는 --to-destination을 쓰는듯?

### firewalld
- 네트워크 연결의 신뢰도에 따라 영역을 구분하여 사용하므로 방화벽 규칙을 효율적으로 관리할 수 있음.
- **영역 확인 명령어**
	- `ls -l /usr/lib/firewalld/zones/`
- **주요 명령어**
	- 이 책에서 firewalld는 세세하게 옵션을 알려주지 않음. 출제가 많이 된 적이 없는건지
	- `firewall-cmd --state` : 상태를 확인
	- `firewall-cmd --reload` : 설정 다시 로드. 영구 옵션 없는 규칙은 사라짐.
	- `firewall-cmd --get-zones` : 존 목록을 출력
	- `firewall-cmd --get-default-zone` : 기본 존 출력
	- `firewall-cmd --get-active-zones` : 활성화된 존 출력
	- `firewall-cmd --add-service=http --add-service=https` : HTTP, HTTPS 서비스 추가
	- `firewall-cmd -add-port=21/tcp` : TCP/21 포트를 추가함
	- `firewall-cmd --zone=public --list-all` : 퍼블릭 존의 사용 가능한 서비스, 포트 목록을 출력한다
	- `firewall-cmd --remove-port=21/tcp` : --zones 옵션이 없으면 기본 존을 대상으로 함. TCP/21 포트 삭제.
	- `firewall-cmd -add-port=8080-8090/tcp` : 포트 범위를 지정하여 추가
	- `firewall-cmd --permanent --add-service=ftp` : FTP를 영구적으로 추가
	- `firewall-cmd --set-default-zone=block` : 기본 존을 block으로 변경. 외부 접근을 다 무시함. reject여서 응답을 보냄.
	- `firewall-cmd --add-forward-port=port=80:proto=tcp:toport=8080` : tcp/80으로 들어오는 패킷을 8080포트로 포워딩함.
	- `firewall-cmd --permanent -add-imcp-block=echo-request` : ICMP Echo-Request에 응답하지 않음.
	- ` systemctl start firewalld` : firewalld 서비스 시작
	- `firewall-cmd --zone=public --add-service=http` : HTTP 서비스 임시 허용
	- `firewall-cmd --permanent --zone=public --add-service=http` : HTTP 서비스 영구 허용
	- `firewall-cmd --permanent --zone=public --add-port=8080/tcp` : 8080 포트 영구 개방
	- `firewall-cmd --list-all` : 현재 영역(zone)의 서비스·포트·규칙 확인
