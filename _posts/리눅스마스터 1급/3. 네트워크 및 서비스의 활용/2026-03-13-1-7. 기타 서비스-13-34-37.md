## 1. 슈퍼 데몬
### 1. 슈퍼 데몬의 개요
- **슈퍼 데몬의 개념**
	- 리눅스의 inetd 방식은 시스템 자원을 효율적으로 사용함. 다른 서비스 실행, 관리를 하는 데몬을 슈퍼 데몬이라고 한다.
	- inetd 방식은 스탠드얼론 방식과 비교하여 처리가 느리지만 제한된 시스템 자원으로 운영하기 효율적
	- inetd 방식은 **/etc/inetd.conf** 설정 파일을 이용해 서비스 사용 여부를 지정하며, 액세스 컨트롤을 위해 **TCP Wrapper**를 사용한다.
- **최근 리눅스의 서비스 관리 방식**
	- 최근에는 xinetd 방식으로 이용하던 많은 서비스들이 단독 데몬이 되거나 systemd에 의한 방식으로 통합됨.
	- systemd는 소켓 기능(온디맨드)을 통해 효율적으로 메모리 관리가 가능함.
	- TCP Wrapper는 xinetd 방식으로 서비스 되거나 **libwrap** 라이브러리를 사용해야 한다. sshd의 경우 TCP Wrapper 사용이 가능.
	- 하지만 위 2개를 지원하는 서비스가 제한적이므로 접근 제한이 필요할 경우 iptables, firewalld를 사용하는걸 권장함.
- **xinetd의 특징**
	- 리눅스 커널 2.4 이후로는 확장된 xinetd를 사용함.
	- 기본 설정 파일 위치는 **/etc/xinetd.conf**
	- `yum -y install xinetd`로 설치하면 된다.


### 2. TCP Wrapper
- TCP Wrapper의 개념
	- inetd 데몬에 의해 관리되는 서비스는 TCP Wrapper를 통해 접근 제어를 할 수 있음.
	- 데몬 이름은 **tcpd**
	- **/etc/hosts.allow** 파일과 **/etc/hosts.deny** 설정 파일을 이용하여 접근 제어를 한다. **hosts**이다!
	- 설정 파일은 allow를 우선 적용하므로 만약 규칙이 중복될경우 deny의 규칙은 무시된다. 그래서 보통 막을 놈들을 먼저 deny에 작성하고 허용할 놈들을 allow에 작성한다.
- **TCP Wrapper 설정 파일**
	- 두 파일 둘 다 설정 방법은 동일. 하나의 줄에 하나의 규칙. 2줄 이상으로 기술할 경우 역슬래시를 이전 줄 마지막에 입력함.
	- 설정 형식은 `[데몬 목록] : [클라이언트 목록] : [셸 명령어]`
	- 서비스 명이 아니라 실행 데몬 이름을 지정하는 것을 명심. telnet이 아니라 in.telnetd 처럼.
	- 예시)
		- `[hosts.allow] ALL : 192.168.9.0/255.255.255.0` : 192.168.9.0 네트워크 대역에 속한 모든 클라이언트 접속을 허가함.

## 2. 프록시 서비스
### 1. 프록시의 개요
- **개념**
	- 클라이언트와 서버 사이에 위치. 요청과 응답 과정에서 데이터를 중개하는 역할을 함.
- **목적**
	- 서버의 데이터를 캐시하여 인터넷 전송 속도를 빠르게 하기 위해 사용.
	- 서버의 가용성, 부하 분산을 위해서도 사용.
	- 추가로 접근 통제, 악성코드 유입 방지 효과가 있음.

### 2. 리눅스 프록시 서버 (squid)
- **squid 서버의 특징**
	- squid는 GPL 라이선스를 따름. 
	- HTTP, FTP, gohper 등 서비스의 데이터 응답속도를 향상시킴.
	- 기본포트는 **3128**
- **설치**
	- `yum -y install squid`
	- `systemctl start squid.service`
- **서버의 설정**
	- /etc/squid/squid.conf에서 설정하면 된다.
	- **주요 옵션 항목들**
		- cache_dir : 캐시 데이터 저장 경로명
			- ufs : squid의 저장포맷
		- http_port : 사용할 포트 지정
		- `acl [별칭] src / dst / port / srcdomain / dstdomain [해당사항]` : acl 구문으로 별칭을 지정한 후 접근 권한을 설정.
			- **http_access allow / deny**로 설정.
		- cache_mem 크기 : 캐시 크기 설정
		- cache_log 경로 : 로그 파일 지정.

## 3. DHCP 서비스
### 1. DHCP 서비스의 개요
- **서비스의 특징**
	- 동적 아이피잖아 이거
	- **BOOTP**라는 프로토콜을 사용함.
	- **UDP**랑 **브로드캐스트** 통신 방식을 이용.
- **서비스 설치**
	- `yum -y install dhcp-server` : 설치 고고

### 2. DHCP 서비스 설정
- **DHCP 서비스 설정 파일**
	- dhcpd(데몬)는 **/etc/dhcp/dhcpd.conf**를 설정 파일로 사용한다.
	- 이때 설정 문장 뒤에는 반드시 **세미콜론**이 있어야 함.
	- DHCP의 로그를 다른 곳으로 전달하려면 **log-facility** 항목을 이용함
	- 특정 호스트에 고정 아이피를 할당할 경우 **fixed-address** 항목을 사용함. 방식은 `host [호스트이름] { 설정 내용 }`
	- 서브넷과 넷마스크도 지정 가능. 지정은 호스트처럼 하면 됨.
	- **주요 설정 항목**
		- range : 아이피 범위
		- range dynamic-bootp
		- option domain-name
		- option domain-name-server
		- **option routers** : 게이트웨이 지정. ㅅㅂ gateway는 없다
		- option broadcast-address 
		- **default-lease-time** : 초 단위로 임대요청 만료시간 설정
		- **max-lease-time** : 초 단위로 클라가 사용할 아이피 최대 시간 지정
		- option subnet-mask
		- **fixed-address** : 고정 아이피 주소 할당
		- 나머지는 알잘딱 되는듯

## 4. VNC 서비스
### 1. VNC 서비스(버츄얼 네트워크 컴퓨팅)의 개요
- **특징**
	- VNC 서비스는 비트맵 이미지 기반의 **RFB** 프로토콜을 이용. GUI 방식으로 원격 컴퓨터에 접속 및 사용하는 기능을 제공.
	- **TCP/5900+(디스플레이 번호)** 를 기본 포트로 사용. 동시에 여러 클라이언트가 접속해 화면 공유가 가능하다.
- **설치**
	- `yum -y install tigervnc-server`

### 2. VNC 서비스의 설정과 실행
- **접속 방법**
	- 세션 공유 / 독립 세션 2가지로 나뉨.
		- 세션 공유 : 로컬과 원격 호스트가 화면, 마우스를 공유함. 암호, 알림 등 정보도 함께 설정이 가능.
		- 독립 세션 접속 : 별도의 세션을 이용함. 환경설정 파일인 **/usr/lib/systemd/system/vncserver@.service**에 관련 항목을 설정함.
- **VNC 서비스 설정**
	- 파일은 **/usr/lib/systemd/system/vncserver@.service**
	- 설정 파일에서는 **/etc/systemd/system/vncserver@.service**로 파일을 복사하라고 함. 그러면 그 파일을 참조하게 됨.
	- `<USER>`를 원하는 사용자 계정명으로 변경한다.
	- `vncpasswd` 명령으로 VNC 서버에 접속할 때 사용할 비번을 설정한다. 패스워드는 사용자 홈 디렉토리 **~./vnc/passwd**에 저장됨.
- **주요 명령어**
	- vncpasswd : 패스워드 설정
	- Xvnc : vncserver에 의해 실행되는 **서버**. 클라이언트 아님
	- vnconfig : 서비스 설정을 관리하고 제어
- **서비스 실행**
	- `systemctl daemon-reload`으로 변경한 설정을 적용.
	- `system start vncserver@:[디스플레이번호].service` 명령으로 VNC 서비스를 실행. 디스플레이 번호가 1번이라면 vncserver@:1이 된다.

### 3. VNC 서비스 사용하기
- **VNC 클라이언트 설치**
	- `yum -y install vnc / tigervnc` 를 사용하면 된다. 
- **실행과 접속 정보 입력**
	- 이후 정보는 서버 주소 : 디스플레이 번호 형식으로 입력하고 비번을 입력하면 끝.

## 5. NTP 서비스
### 1. NTP 서비스의 개요
- **특징**
	- 네트워크 타임 프로토콜의 약자. 
	- 컴퓨터 간의 시간을 동기화 한다.
	- UTC를 기준으로 함. 1/1000초까지 시간을 동기화 가능.
	- 포트는 **UDP/123**번을 사용.
- **NTP의 계급 구조**
	- NTP는 클럭소스 수준의 계층적, **반 계층화** 시스템을 사용. 여기서 각 계층 구조의 수준을 계급이라고 함.
	- 원자시계 도는 GPS와 같은 장치인 0단계부터 가장 낮은 단계인 15단계의 계층으로 구성됨.
		- **tum 16**은 비동기 장치를 의미함.
	- 숫자가 낮은 단계가 높은 단계보다 품질이 우수함.

### 2. NTP 서비스의 설치와 실행
- **설치와 실행**
	- `yum -y install ntp`로 설치.
	- `systemctl start ntpd.service`로 실행.
- **서비스의 설정**
	- **/etc/ntp.conf**를 설정 파일로 이용.
	- 주요 설정 항목
		- driftfile
			- NTP 데몬에 의해 자동으로 생송되는 driftfile의 위치를 지정함.
			- 이 파일은 시간 오차의 평균값을 저장하여 시간을 정확하게 유지하는 역할을 함.
			- ex) driftfile /var/lib/ntp/drift
		- restrict
			- NTP 서버에 접근할 수 있는 클라를 제한함.
			- ex) `restrict 192.168.1.0 mask 255.255.255.0 nomodify notrap`
		- server
			- NTP 서버를 지정.
		- keys
			- 대칭키 암호화를 위한 키 파일을 지정.
			- ex) `keys /etc/ntp/keys`

### 3. NTP 서비스의 명령어
- **주요 관련 명령어**
	- ntpdate
		- `ntpdate [서버주소]`를 사용하면 원격 서버와 시간을 동기화함. 
		- -d 옵션을 사용하면 서버와 시간 차이를 확인할 수 있음. 단 로컬 시간을 업데이트 하지 않음. (디버깅 모드)
	- ntpq
		- NTP 질의(쿼리) 명렁어.
		- -p 옵션으로 연결된 서버의 상태를 출력
	- 질의 명렁어 목록은 help로 확인하면 된다.
	- 만약 설치에 문제가 있을 경우 **chrony**를 사용하면 된다.
		- 최근 리눅스는 이걸 기본으로 사용하는 편

### 4. chrony의 설치와 활용
- **설치와 실행**
	- `yum -y install chrony`
	- `system start chronyd / sudo systemctl enable chronyd`
- **chrony 서비스의 설정**
	- **/etc/chrony.conf**를 설정 파일로 이용함.
	- 주요 설정 항목
		- pool
			- 공개된 서버들의 목록을 포함할 풀을 지정. 
		- startumweight
			- 계급당 보정해야할 거리 값 지정
		- driftfile
			- 클럭 보정 정도를 저장하는 파일 지정.
		- makestep
			- 시스템의 시간이 크게 빠르거나 느릴 때 조정하는 방식.
			- ex) `makestep 1.0 3` : 시스템 시작 후 처음 3회 시간 동기화에서 시스템의 시간과 NTP 서버의 시간 차이가 1초 이상일 경우 즉시 조정
		- allow : 서버에 접근 가능한 클라 주소 범위 지정
		- keyfile : 인증 시 사용할 키 파일 지정. 기본은 **/etc/chrony.keys** 인듯
		- logdir : 로그 파일 경로. 기본은 **/var/log/chrony** 인듯
	- 만약 컨픽을 건드렸다면 재시작해주자.
- **chronyc 명렁어의 활용**
	- chronyc 명령어를 사용하여 서비스를 제어 가능.
	- `sudo chronyc source -v` : NTP 소스 정보 확인
	- `sudo chrony -a makestep` : 즉시 시간 동기화
	-  `sudo chronyc tracking` : 시간 동기화를 확인
