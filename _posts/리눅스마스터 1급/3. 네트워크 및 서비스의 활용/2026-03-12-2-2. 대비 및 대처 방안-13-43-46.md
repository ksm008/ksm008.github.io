## 1. 보안 솔루션을 이용한 대비 및 대처
### 1. 방화벽
- 1. 개념과 주요 기능
	- 외부로부터 들어오는 나쁜 트래픽을 막고 허가된 트래픽만 내부로 유입될 수 있도록 통제함.
	- 내부에서 외부, 외부에서 내부 둘 다 룰에 따라 통제 가능.
	- 접근제어 (Access Control), 로깅, 감사 추적 (Auditing), 사용자 및 호스트 인증 (Authentification) 기능 등을 제공한다.
- 2. 종류
	- **스크리닝 라우터 (Screening Router)**
		- 내/외부 네트워크를 방화벽이 탑재된 라우터로 연결함.
		- 필터링 속도가 빠르고 네트워크 전체를 방어할 수 있음.
		- 패킷 필터링 규칙 구성, 검증 및 로그 관리가 어려움.
	- **바스티온 호스트 (Bastion Host)** 
		- 방화벽 기능을 탑재한 *호스트*를 이용하여 내/외부 네트워크를 연결
		- 호스트 기반의 소프트웨어로 지능적인 정보 분석을 지원
		- 만약 호스트 자체에 문제가 생기면 큰일남.
	- **듀얼홈 게이트웨이 (Dual-Homed Gateway)**
		- 2개의 네트워크 가드를 내장한 바스티온을 사용. 
		- 네트워크가 분리가 되어 내부 IP 주소가 노출이 안돼용 -> 보안성이 높다
		- 이것도 호스트가 병신이면 문제가 된다
	- **스크린드 호스트 게이트웨이 (Screend Host Gateway)**
		- 듀얼 홈 게이트웨이와 스크리닝 라우터를 합침.
		- 당연히 보안성이 높겠죠?
		- 하지만 구축 비용이 높고 라우팅 테이블 변조 공격에 취약함
	- **스크린드 서브넷 게이트웨이 (Screend Subnet Gateway)**
		-  스크리닝 라우터를 바스티온 호스트 양쪽에 연결함
		- 바스티온 호스트는 **프록시 서버**를 사용하여 허용되지 않은 모든 트래픽 거부
		- 이 정도 하면 보안은 뭐 당연한거고
		- 비용 높은 것도 당연한거고 서비스 응답성이 좀 느려질 수 있다는 것도 당연한거고
		- 아니 그냥 계속해서 짬뽕시키는거구만 

### 2. 침입 탐지 시스템 (Intrusion Detection System, IDS)
- 1. 개념 및 주요 기능
	- 탐지하는 애겠죠 뭐겠음..
	- 데이터 수집, 침입, 탐지, 추적 및 보고를 한다.
- 2. 시스템 유형
	- 데이터 소스를 기준으로 호스트 기반이냐, 네트워크 기반이냐를 구분할 수 있음.
		- **HIDS** : 서버에 직접 설치되어 침입 탐지를 수행한다.
		- **NIDS** : 네트워크 세그먼트 별로 탐지 장비를 설치하여 운영.
	- 침입 탐지 방식을 기준으로 오용 탐지냐, 이상 탐지로 구분할 수 있음.
		- **오용 탐지 (Misuse)** : 알려진 공격 패턴을 등록하고 탐지함. 오탐율이 낮지만 알려지지 않은 공격에는 취약함.
		- **이상 탐지 (Anormaly) :** 정상 패턴이랑 좀 다르거나 이상하면 공격이라고 판단함. 알고리즘을 사용하여 새로운 공격 패턴에 대응할 수 있지만 오탐할 수도 있다.

### 3. 침입 방지 시스템 (Intrusion Prevention System, IPS)
 - 1. 개념
	 - 수동적인 방어가 아니라 능동적인 방어랑 차단 등 사전 공격대응을 중심으로 한 시스템이래요... 뭐...
	 - 공격 자체를 차단한다고 함

## 2. Snort를 이용한 대비 및 대처
### 1. Snort의 개요
- 1. 특징
	- 리눅스 시스템에서 쓸 수 있는 공개형 IDS.
	- 크게 스니퍼 (Sniffer), 프리 프로세서 (Pre-processor), 탐지 엔진, 로깅 (출력)으로 구성되어있다.
- 2. 주요 기능
	- 탐지 룰을 사용하여 네트워크 트래픽을 분석하고 침입을 탐지할 수 있다.
	- 주요 기능은 아래와 같다.
		- **패킷 스니퍼** : 네트워크의 패킷을 읽고 표시한다.
		- **패킷 로거** : 로깅함
		- **네트워크 IDS** : 네트워크 패킷을 분석하여 공격 탐지. 버퍼 오버플로우, 포트 스캔 등 다양한 공격 감지 가능.

### 2. Snort의 룰 설정
- 1. 구조
	- 크게 헤더와 룰 옵션으로 구분됨.
		- **헤더 :** `[동작 방식 (액션)][프로토콜][소스 아이피와 포트][흐름 방향 ( -> 화살표)][목표 아이피와 포트]`
		- **룰 옵션** : General, Payload, non-payload, post-detection 4가지로 분류 가능. 이중 payload 탐지 룰 옵션을 주로 사용함.
		- **룰 옵션의 주요 항목**
			- msg : 출력할 메세지 지정
			- sid: 룰은 1000000 이상으로 함.
			- content : 페이로드 내부에서 검색할 문자열 지정
			- depth : 검사할 바이트 설정
			- offset : 검사 위치 설정
			- distance : 검색할 문자열이 여러가지일 경우 다음 검색하는 문자열을 해당 바이트 이후부터 검색함
			- within : 지정한 바이트 수 내에서만 검사
			- nocase : 대소문자 구분 안함
			- sameip : 소스와 목표 주소가 동일한 것을 검사
	- 예시)
	- `alert tcp any any -> 192.168.10.0/24 80 (msg:"passwd detected"; content:"passwd"; nocase; sid:1000001;)` : 외부에서 내부 80 포트로 이동하는 패킷중 passwd 문자열이 포함된 패킷이 발견되면 "passwd detected"라는 경고 메시지를 발생시킴.
	- **랜드 어택**은 sameip로 손쉽게 체크할 수 있을 것.

> **Suricata 도구**<br>
> - Snort를 베이스로 개발되고 룰이 호환된다.
> - 멀티 코어, GPU 가속화를 지원하여 처리 속도가 빨라용 대충 좋다는 뜻

## 3. iptables를 이용한 대비 및 대처
### 1. iptables의 개요
- 1. 특징
	- 리눅스의 방화벽임. 패킷 필터링을 통해 분석, 허용, 차단이 가능함. 또한 NAT (Network Translation Address)에도 사용이 가능함.
	- 커널의 넷필터 (Netfilter) 모듈을 이용하여 네트워크 패킷을 필터링 한다.
	- 정책은 총 4가지가 있다.
	- 일부 패킷 허용 / 제한 방식
	- 모든 것을 허용 / 거부하는 방식

> **firewalld**<br>
> 최근 리눅스는 firewalld를 기본으로 사용하고 iptables보다 편리하다. 하지만 이게 더 명확한 규칙을 설정할 수 있으므로 여전히 중요함. firewalld도 iptables를 기반으로 동작한다. 
> - `systemctl stop firewalld`
> - `systemctl disable firewalld`
> - `yum -y install iptables-services`
> - `systemctl start iptables`
> - `systemctl enable iptables`
> - `systemctl -l status iptables`
{: .prompt-info }

### 2. iptables의 테이블 구조
- 1. 테이블 종류
	- 총 5가지의 테이블을 지원함.
	- **filter :** iptables의 기본 테이블. 패킷 필터링 기능 담당.
	- **nat :** IP 주소 및 포트를 변환, 관리를 함 (Network Address Translation). 하나의 공인 IP를 여러 호스트가 사용할 때 주로 사용
	- **mangle :** 성능 향상을 위한 TOS 설정과 같이 패킷 데이터를 변경하는 특수 규칙을 적용.
	- **raw :** 연결추적을 위한 세부 기능을 제공.
	- **security :** SELinux(보안커널)에서 사용하는 접근제어 규칙을 적용.
- **2. filter 테이블**
	- 미리 정해진 3가지 체인이 있다. 이를 이용하여 네트워크 패킷을 대상으로 정해진 규칙을 수행하고 패킷 필터링, 방화벽 기능을 수행한다.
		- **INPUT :** 유입되는 패킷을 필터링
		- **OUTPUT :** 나가는 패킷 필터링
		- **FORWARD :** 내가 라우터일때 나를 통과하는 패킷. 
- 3. nat 테이블
	- 4가지 체인이 존재.
		- **PREROUTING :** 패킷의 도착지 주소를 변경
		- **POSTROUTING :** 출발지 주소를 변경하며 마스커레이드라고도 함. (masquerade)
		- **OUTPUT :** 호스트에서 밖으로 흐르는 도착지 주소를 변경
		- **INPUT :** 호스트 밖에서 안으로 흐르는 패킷의 주소를 변경.


### 3. iptables 사용하기
- 1. 사용법
	- `iptables [-t 테이블 이름] [액션] [체인 이름] [매치 규칙] [-j 타깃]` 형식을 이용.
	- **체인을 대상으로 한 액션의 항목**
		- -N : 새로운 정책 체인을 만듬. new.
		- -X : 인풋, 아웃풋, 포워드를 제외한 비어있는 정책 체인을 제거함. execute의 약자라고 생각하기
		- -L : 현재 정책 체인 목록을 표시. 리스트.
		- -F : 지정된 체인에 설정된 모든 정책을 삭제. 플러시.
		- -C : 패킷을 테스트함. 체크의 약자인듯
		- -P : 체인의 기본 정책을 설정. 폴리시.
		- -Z : 체인 내의 모든 규칙들의 패킷, 바이트 카운터를 0으로. 제로의 약자
	- **체인 내부를 대상으로 한 주요 액션 항목**
		- -A : 어펜드. 새로운 정책을 가장 마지막에 추가 등록.
		- `-I [체인] [라인번호]` : 지정한 라인 번호에 추가. 인서트.
		- 그 외에 딜리트, 리플레이스 약자인 -D, -R도 위와 동일함.
	- **주요 매치 규칙 항목**
		- -s : 출발지 ip, 도메인 설정
		- -d : 목적지 ip, 도메인 설정
		- ! - 뒤 따라오는 규칙은 제외.
		- -p : tcp, udp, icmp등 프로토콜 설정
		- -i : 입력 네트워크 인터페이스 설정
		- -o : 출력 네트워크 인터페이스 설정
		- --sport : 출발지(소스) 포트 설정, ':'을 사용. 목적지는 당연히 dport겠죠?
		- --tcp-flags : SYN, ACK등 TCP flag를 지정함
		- --syn : TCP flag로 SYN만 가진 것을 지정
		- --icmp-type : ICMP 타입을 지정함. echo-request(8), recho-reply(0), destination-unreachable(3)등이 있음.
		- -m : 특정 모듈, 규칙과 매치함.
		- --state : 연결 상태와 매치함
		- --string : 응용 계층의 데이터 바이트 순서와 매치함
	- **기타 부가 옵션 항목**
		- -n : 출력 정보를 숫자로 표시
		- -v : 부가 정보를 표시
		- --line-numbers : 정책 앞에 번호를 붙여서 표시
	- **-j 타깃에 사용할 수 있는 항목**
		- ACCEPT : 패킷 허용
		- REJECT : 패킷 거부. TCP일 경우 reset, UDP일 경우 Port unreachable 응답 메시지 전송.
		- DROP : 패킷 거부하고 버림. 응답 메시지 없음.
		- LOG : 패킷을 syslog에 전달하여 기록. 경로는 **/var/log/message**
		- RETURN : 호출 체인 내에서 패킷을 계속 진행함.
- **2. 필터링 예제**
	-
