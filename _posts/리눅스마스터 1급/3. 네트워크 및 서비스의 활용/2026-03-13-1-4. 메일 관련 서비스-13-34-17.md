## 1. 메일 관련 서비스 개요
- **관련 프로토콜**
	- SMTP : TCP 25번 포트, 메일 전송 프로토콜.
	- POP / POP3 : TCP 110번, 메일 수신 프로토콜. 메일을 가져온 후 삭제함.
	- IMAP : TCP 143번, 메일 수신 프로토콜. 이메일 메시지를 서버에 남겨두었다가 나중에 삭제할 수 있음.
- **관련 프로그램**
	- MTA : SMTP 프로콜을 사용하여 메일 전송 
	- MDA : procmail이 대표적 프로그램. 대리인 역할을 수행하는 프로그램. 메일을 대행해서 가져오거나 전달함.
	- MUA : 사용자가 메일 수신, 발신할 때 사용. kmail, evolution, mutt 가 있음.

## 2. 메일 관련 서비스 사용
- **설치**
	- `yum -y install sendmail`
- **주요 설정 파일**
	- /etc/mail/sendmail.cf
		- 기본 설정 파일
	- /etc/mail/sendmail.mc
		- 설정을 편하게 관리할 수 있는 보조 파일.
	- /etc/aliases
		- 별칭 혹은 특정 계정으로 수신한 이메일을 다른 계정에 전달하도록 도와줌.
	- /etc/mail/access
		- 접근을 제어함
	- /etc/mail/virtusertable
		- 가상 메일 계정 관련
	- /etc/mail/local-host-names
		- 메일 수신지를 설정함. 다시 시작해야 적용됨
	- ~/.forward
		- 다른 메일로 포워딩할 때 설정하는 파일
- **/etc/mail/sendmail.cf 설정**
	- Cw : 메일 수신 호스트 설정. 보통 도메인 명.
	- Fw : 여러 개의 도메인명을 수신 호스트의 이름으로 이용할 경우...
	- Ft : 신뢰하는 유저.
	- Dj : 도메인 이름 강제
	- Dn : 회신하는 사람 이름 지정.
	- FR -o : 릴레이 허용할 도메인.
	- Kvirtuser : 가상 호스트 동일 계정 공유
	- Kaccess : 접근 제어 담당
	- O AliasFile : 다른 계정으로 전달할 수 있도록 설정 파일 지정
	- O MaxMessageSize
- **관련 주요 명령어**
	- sendmail
		- -pq : 메일 큐의 상태 표시
		- -bi : aliases 정보 갱신
		- -oQ : 특정 큐의 상태 표시
	- mailq : 메일의 큐를 표시함.
		- -Ac 옵션을 붙여 /etc/mail/submit.cf 파일의 지정된 큐의 상태를 표시함.
		- /var/spool/clientmqueue 큐의 상태를 표시.

## 2차 시험 대비하면서 다시 정리
- 메일 관련 서비스 패키지 이름은 **sendmail**
- **주요 설정 파일 << 여기 부분이 출제 확률이 높음**
	- **/etc/mail/sendmail.cf** 
		- 기본 설정 파일
	- **/etc/mail/sendmail.mc** 
		- sendmail의 설정을 편리하게 할 수 있는 보조 파일.
		- m4 유틸리티를 m4 > sendmail.mc > sendmail.cf와 같이 사용하여 sendmail.cf를 생성한다.
		- sendmail-cf 패키지가 필요.
	- **/etc/aliases**
		- 메일의 별칭, 혹은 특정 계정으로 수신한 이메일을 다른 계정으로 전달하도록 설정. 보통 여러 사람에게 전달할 때 사용함.
		- `[수신 계정] : [전달 계정]` 형식을 따른다. 
		- `:include:[파일 이름]`으로 사용자 이름이 저장된 파일을 설정할 수 있다.
		- sendmail이 참조하는 파일은 /etc/aliases.db이므로 /etc/aliases를 수정한 후 newaliases나 sendmail -bi 명령으로 적용한다.
	- **/etc/mail/access** 
		- 메일 서버에 접속하는 호스트의 접근을 제어하는 설정 파일. 스팸 메일 방지 등에 사용 가능.
		- `[정책 대상][정책]`의 형식을 사용. 
		- 정책 대상은 도메인명, 아이피, 메일 주소를 사용. 
		- 정책은 아래와 같다.
			- RELAY (릴레이 허용)
			- DISCARD (거부)
			- 거부 후 메시지 전송 (REJECT)
			- DNS 조회 실패 시에도 허용 (OK)
		- 위 정책을 지정할 수 있다.
		- `makemap hash /etc/mail/access < /etc/mail/access` 같은 명령으로 **/etc/mail/access.db**에 적용함.
	- **/etc/mail/virtusertable**
		- 가장의 메일 계정으로 들어오는 메일을 특정 계정으로 전달하는 정보를 설정.
		- 다 작성 후 `makemap hash /etc/mail/virtusertable < /etc/mail/virtusertable` 와 같은 명령으로 **/etc/mail/virtusertable/db**에 적용한다.
	- **/etc/mail/local-host-names**
		- sendmail에서 수신할 메일의 도메인과 호스트, 즉 메일 수신지를 설정하며 sendmail을 다시 시작하여 적용한다.
	- **~/.forward**
		- 사용자 개인이 수신한 메일을 다른 메일로 포워딩할 때 설정하는 파일. 원하는 메일 주소를 한 줄씩 추가함
- **/etc/mail/sendmail.cf** 파일 건드리는건 웬만해서 안나오는 것 같다. 전날에나 좀 볼 예정.
