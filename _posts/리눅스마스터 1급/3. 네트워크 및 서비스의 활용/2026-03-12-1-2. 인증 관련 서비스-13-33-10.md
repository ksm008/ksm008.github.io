## 1. 인증 관련 서비스의 개요
### 1. 리눅스 인증의 개요
- 1. 인증과 인가의 개념
	- 인증은 사용자가 시스템에 등록되어 있는지 확인함.
	- 인가는 권한이 있는지를 확인함.
- 2. 리눅스의 기본 인증 방법
	- /etc/passwd, /etc/shadow를 가지고 하겠죠?
- 3. 네트워크 기반 인증 서비스의 필요성
	- 하나의 시스템에 적은 인원이 대상이면 다수의 시스템을 운영할 경우 어려워진다.
	- **NIS** (네트워크 인포메이션 시스템)랑 **LDAP** (라이트웨이트 디렉터리 액세스 프로토콜)가 있다.

### 2. NIS와 LDAP 서비스의 주요 특징
1. NIS 서비스의 주요 특징
	- 썬사가 개발함.
	- 호스트명, 사용자명, 사용자 암호 등과 같은 시스템 정보를 검색하고 관리하기 위한 서비스. RPC(리모트 프로시저 콜)를 사용함.
	- NIS 서버에 등록된 사용자 계정, 암호 그룹 정보 등을 네트워크를 통해 다른 시스템에게 제공함으로 여러 호스트들이 동일한 계정 정보를 사용 가능.
	- NIS를 사용해 텔넷, 삼바, ssh등 서비스에서 사용자 인증이 가능함.
2. LDAP 서비스의 주요 특징
	- 디렉터리 서비스를 조회하고 수정하는 TCP 기반 응용 프로토콜
	- **X.500 DAP**를 기반으로 함.
	- RDBMS에 비해 검색은 빠르지만 자주 변경되는 정보 관리에는 불리함.
	- LDAP 서버에는 여러 엔트리가 트리 구조로 되어있음.
	- **주요 속성**
		- c : 국가 이름.컨트리
		- st : state. 우리나라는 도, 외국은 주.
		- I : 도시, 지역.
		- street : 도로명주소
		- dn : 조직 내 고유한 식별자
		- rdn : 상대방의 dn
		- o : organization. 조직 이름
		- ou : 부서 이름. unit인가?
		- cn : common name. 전체 이름
		- sn : 성
		- givenName : 이름
		- dc : 도메인 네임 요소
		- mail : 이메일
		- telephoneNumber : 전화번호
	- 각 엔트리는 DIT라는 트리 구조로 조직화되고 DN이라는 고유 식별자로 지칭된다. 
	- 마이크로소프트의 Active Directory는 LDAP을 기반으로 인증 서비스를 제공함.

## 2. NIS 서비스 사용하기
### 1. NIS 서버 설치와 구성
- RPC를 사용하므로 사전 작업을 해야한다.
	- systemctl start rpcbind
	- systemctl enable rpcbind
- 이후 /etc/hosts 파일에 아이피랑 도메인 등록. 미리 등록을 해두면 외부 DNS 쿼리 없이 도메인과 아이피 매핑을 사용 가능
- 그리고 네트워크 서비스를 다시 시작. `systemctl restart NetworkManager`
- 그리고 서버 패키지 이름은 ypserv.
	- `yum -y install ypserv`로 설치하기.
- **NIS 도메인 설정**
	- nisdomainname 아니면 /etc/sysconfig/network에 수동 등록. 여기에 등록하면 재부팅시에 자동 적용됨.
	- nisdomainname로 설정한 도메인 명을 확인할 수 있다.
- 그리고 NIS 클라이언트용 계정을 생성.
- 그리고 관련 데몬 3개 다 시작. ypserv, yppasswdd, ypxfrd.
- 이후 정보를 갱신해야 될 때는 **/var/yp** 경로로 이동하여 make를 실행하거나 `make -c /var/vp`를 사용하면 됨.
- make를 완료하면 해당 디렉터리에 NIS 서버 이름의 디렉터리와 NIS 정보를 가지고 있는 맵 파일이 생성됨. 만약 새로운 사용자를 추가하면 해당 맵 파일도 갱신해야된다.
- `systemctl restart, enable`을 사용하여 전부 재시작, 리부트 등록을 하면 된다.

### 2. NIS 클라이언트 설치와 구성
- `yum -y install ypbind yp-tools` : 이 2개를 받아야 함.
	- ypbind는 바인드 정보를 관리하고 NIS 서버, 클라이언트를 서로 연결함.
- 또 nisdomainname을 사용하거나 /etc/sysconfig/network에다가 수동 등록 하기.
- 그리고 **/etc/yp.conf** 파일 에다가 NIS 서비스와 도메인 정보를 설정함.
- 마지막으로 클라이언트 데몬인 ypbind를 시작.
	- `systemctl start ypbind.service`
- 이후 ssh를 사용하여 client 호스트에 로그인하고, 계정 아이디랑 패스워드는 NIS에 등록한것을 사용하면 된다.
	- `ssh -l nisuser 192.168.45.216`
- 일반적으로 NIS 서버만 동작시킨 상태에서는 사용자 디렉터리가 생성되지 않지만 authconfig에서 설정할 수 있다.

### 3. NIS 관련 주요 명령어
- nisdomainname
- ypwhich
	- NIS를 사용해 로그인한 후에, 인증에 사용한 NIS 서버를 조회함. -m을 통해 맵 조회 가능
- ypcat
	- NIS 서버의 구성파일을 확인할 수 있다. 
- yptest
	- 클라이언트에서 동작 및 설정 등을 확인하고 도메인 명, 계정 정보 등을 출력함.
- yppasswd : 서버에 등록된 사용자 비번 변경
- ypchsh : 사용자 셸 변경
- ypchfn : 사용자의 정보를 변경


## 2차 시험 요약
- NIS 서버의 패키지 이름은 **ypserv**
- NIS 클라이언트의 패키지는 2개. **ypbind**와 **yp-tools**
	- ypbind는 바인드 정보를 관리하며 NIS 서버와 클라이언트를 연결함
- **nisdomainname :**  NIS 도메인명 설정
	- `nisdomainname test-domain.com`
- **/etc/sysconfig/network** 파일에 NIS 도메인을 등록하면 재부팅시에도 자동 적용 가능
```bash
vi /etc/sysconfig/network
...
NISDOMAIN=test-domain.com
```
- **주요 서비스 파일들**
	- **ypserv.service** : 메인 데몬
	- **yppasswdd.service** : 사용자 패스워드 관리
	- **ypxfrd.service** : 서버와 클라이언트 사이의 맵핑 속도를 높여줌
- 사용자 추가 등 NIS 관련 설정 이후 정보를 갱신하려면 **/var/yp** 디렉터리로 이동 후 **make**를 실행하거나 `make -c /var/yp` 명령을 사용하자.
- ... 나머지는 내일 작성하자
