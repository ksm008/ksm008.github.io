## 1. 사용자의 분류

루트, 시스템, 사용자 계정으로 나뉨. 
- 루트는 모든 권한. 대빵임.
- 시스템 계정은 제한적임. 첨부터 생성됨. 

## 2. 루트 계정 관리
### 1. 루트 계정 설명
- 수퍼 유저라고 불림. 제한 없이 어느 명령어든 사용이 가능. 보통 시스템 대표가 루트를 관리함. UID는 0으로 고정.

### 2. 루트 계정 권한 획득 방법
- 기본적으로 루트는 로그인 못함. `sudo passwd root`를 통해 비번 설정하고 얻음.
- `su` 또는 `su - `로 전환.
- 한번 로긴하면 모든 작업 제약 없이 가능. 
- 아니면 `sudo`를 명령어 앞에 붙여서 권한을 임시로 얻을 수도 있다.
- /etc/passwd 파일에서 사용자 UID를 0으로 바꿔도 루트 권한을 얻을 수 있다.

### 3. 루트 계정 관리 방안
- 초기 환경설정에만 사용하고 이후는 ssh 통해서 로긴 안되게 설정해야함.
- 루트는 유일해야 하므로 다른 사용자 uid가 0인지 확인해야함.
- PAM을 통해 로긴 불가능하게.
- 부득이하게 루트로 로긴하더라더 TMOUT 환경변수로 자동 로그아웃되게 설정.
- 되도록 sudo를 쓰자.

## 3. 시스템 계정 관리
### 1. 시스템 계정 설명
- 메일, SSH 연결 등 특정 서비스 권한을 행사함.
- bin, daemon, adm, lp, sync 등 계정들이 시스템 계정. /etc/passwd에서 uid가 1 ~ 499 사이인 놈들임.

## 4. 사용자 계정 관리
### 1. 설명
- 사용자 계정이라고
- 그룹에 속할 수 있고 접근 권한 뭐 이런게 된대요 당연한거...
- 레드햇 이전은 uid가 500부터, 하지만 7, 오픈수세나 데비안 버전은 1000이상 값을 가짐.
- 일반 사용자 계정 uid 범위는 **/etc/logins.defs**에 정의되어있다!

### 2. useradd
- 옵션이 뒤지게 많아요. 소괄호 안은 -- 사용.
	- -d : 홈 디렉터리 (home)
	- -D : 생성시 사용하는 기본값 보기 또는 설정 (defaults)
	- -e : 계정 만기일 지정. (expiredate)
	- -f : 패스워드 만료일. 숫자만큼 패스워드가 유효하다. (inactive)
	- -g : 그룹 추가
	- -m : 홈 디렉터리를 지정하고 없으면 생성. (create-home)
	- -k : -m 옵션으로 홈 디렉터리 생성할 때 복사할 기본 파일을 지정 (skel SKEL_DIR)
	- -M : 홈 디렉터리 생성하지 않음.
	- -N : 사용자와 동일한 이름으로 홈 디렉터리 생성하지 않음 (
	  --no-user-group)
	- -p : 패스워드 설정 (--password, PASSWORD)
	- -r : 시스템 계정 생성. -m 옵션 안쓰면 홈디렉터리 안만듬 (system)
	- -s : 사용할 셸 지정. (--shell, SHELL)
	- -u : uid 값 지정. (--uid, UID)

### 3. passwd
- 패스워드 설정하는거
	- -d : delete. 비번 삭제
	- -e : expire, 비번 만료. 다음 로긴에 비번을 변경해야함.
	- -i : inactive, 계정 비활성화 유예 기간을 일수로 지정.
	- -l : lock, 락. 로그인을 막음.
	- -n : mindays, 며칠 후에 비번 변경이 가능한지 설정
	- -q : quiet, 화면 출력 없이 명령 수행
	- -S : status, 비번 관련 정보 출력
	- -u : unlock, 락 해제.
	- -w : warndays, 만료 전 경고 날짜 지정
	- -x : maxdays, 패스워드 최대 사용 기간 설정
- 루트 사용자는 다른 사용자 패스워드 변경 가능.

### 4. su 
- 사용자를 변경함
	- -c : command, 사용자로 셸이 실행할 명령어 지정
	- -, -l : login, 로그인했을때 환경변수 설정 및 홈 디렉토리로 이동
	- -s : shell, 로긴하고 명시된 셸 사용.

### 5. usermod
- 사용자 셸, 홈 디렉터리, 그룹, uid, gid 변경.
	- -a : append, 그룹을 추가하기 위해 -G 옵션과 함께 사용
	- -c : 사용자에 대한 간단한 정보를 입력. 코멘트.
	- -d : --home. 사용자의 홈 디렉터리를 변경함. 디렉터리로 외우면 될 듯
	- -e : 계정 만료일자. 
	- -f : 패스워드 만료 이후 며칠동안 유효할 것인지를 설정
	- -I : 사용자의 로그인 이름을 변경.
	- -L : 락을 걸어서 로그인을 막음...
	- -m : 사용자의 홈 디렉터리 내용을 새로운 홈 디렉터리로 옮김.
	- -p : 암호화된 패스워드를 설정

### 6. userdel
- 계정 삭제
	- -r :  홈 디렉터리랑 관련 메일 모두 삭제
	- -f : 로긴되어있어도 삭제

### 7. chage
- 사용자의 패스워드 만료 정보 설정. 왜 있는거임? sudo 써야함.
	- -d : 패스워드를 변경해야 할 날짜 수 지정.
	- -E : 계정이 만료되는 날을 설정.
	- -I (대문자 i) : 계정 만료 후 패스워드가 비활성화될 때까지 유예기간을 설정한다.
	- -l : 계정의 만료 정보 보기
	- -m : 최소 날짜 지정
	- -M : 최대 날짜 지정
	- -W : 경고 메세지를 보여줄 날짜 지정

## 5. 그룹 계정 관리
### 1. 그룹이란
- 그룹이 그룹이지 뭐겠냐

### 2. groupadd
- 그룹 생성
	- -f : force. 이미 있는 그룹이여도 성공으로 간주함.
	- -r : 시스템 그룹. system.
	- -g : gid 지정.
	- -o : non-unique. 중복 uid 허용.

### 3. groupmod
- 그룹 정보 변경
	- -g : 변경할 gid 설정.
	- -n : 변경할 이름 설정

### 4. groupdel
- 그룹 삭제

### 5. gpasswd
- 그룹 패스워드 설정
- 패스워드 정보는 **/etc/group**과 **/etc/gshadow**에 저장되어있다!
	- -a : add user, 그룹에 사용자 추가. 아니 뭔 패스워드 옵션에 사용자 추가를 넣어둠?
	- -d : delete user. 그룹에서 사용자 삭제
	- -r : 그룹 패스워드 제거
	- -R : restrict group. 제한 걸기
	- -A : administartor user, 관리자 설정
	- -M : member user, 그룹 멤버 설정

### 6. newgrp
- 현재 로긴중인 사용자의 그룹 변경
- newgrp colors 이런 느낌. 별별명...

## 6. 사용자 환경 설정 파일
- 사용자 계정과 그룹 정보는 전부 /etc 디렉터리 이하에 존재함.
	- /etc/passwd, /etc/shadow : 사용자 계정의 아이디 등 계정 정보, 패스워드랑 정책 설정.
	- /etc/group, /etc/gshadow : 위랑 똑같지만 그룹용.
	- etc/default/useradd : useradd시 기본 설정
	- /etc/login.defs : 로그인 수행 시 기본 설정. 일반 계정 사용자 uid 범위 존재.
	- /etc/skel : 홈 디렉터리 생성시 기본 제공 파일 설정

### 1. /etc/passwd
- 일반적으로 모두 열람이 가능함. 
- 하지만 쓰는건 루트나 권한이 있어야 함.

```bash
cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
...
```

  - 구조는 다음과 같다. 
	  - 사용자명 : 비밀번호 : uid : gid : 사용자 설명 : 홈 디렉터리 : 셸
		  - 비번의 경우 x이면 shadow에 보관중이라는 뜻. 

### 2. /etc/shadow
- 구조는 다음과 같다.
	- 이름 : 암호화된 해시값 : 마지막 변경일 : 패스워드 최소 사용일 : 최대 사용일 : 만료 경고일 : 유예기간 : 만료일
		- 패스워드의 경우 `$id$salt$hashed` 형태. id는 암호화 알고리즘, salt는 소금값, hashed는 패스워드 해시값.
		- 만약 해당하는 값이 없으면 비어있다.
		- 변경일, 만료일의 경우 1970년 1월 1일을 기준으로 계산한다. 왜 그런거지?
		- 패스워드 최대 사용일이 99999면 변경하지 않아도 된다. 

> **$id 의미**<br>
> - `$1` : MD5
> - `$2a, $2y` : Blowfish
> - `$5, $6` : SHA-256 / 512
{: .prompt-info }

### 3. /etc/default/useradd
- 직접 열어보거나 useradd -D로 확인 가능.
- 대충 보면 암.

### 4. /etc/login.defs
- 쉐도우 패스워드 스위트를 읽기 위한 전용 환경설정 파일.
- 뭐 이것도 대충 보면 암.

### 5. /etc/group
- 그룹 사용자 관리 파일.
- 구조는 다음과 같다.
	- 그룹명 : 패스워드 : gid : 멤버 목록
		- 역시 패스워드가 x이면 gshadow에 있겠죠?

### 6. /etc/gshadow/
- 얘는 일반 사용자는 못읽음. 루트만 가능.
- 구조는 다음과 같다.
	- 그룹명 : 패스워드 : 그룹 관리자 : 멤버 목록
		- 패스워드 형식은 그냥 shadow랑 같이 해시 알고리즘, 소금값, 해시값으로 구성됨

## 7. 사용자 및 그룹 정보 관련 명령어
### 1. users
- 현재 로긴 중인 모든 사용자 출력

### 2. who
- 똑같이 로긴된 모든 사용자 출력이긴 한데 터미널 종류나 로그인 시각 등이 기본 출력됨. 옵션이 많음. 개같은거
	- who am i : -m 옵션과 동일. 현재 로그인한 사용자 정보 출력
	- -aH : 모든 정보를 각 컬럼에 헤더와 함께 출력한다.

### 3. whoami
- 현재 로그인한 사용자명만 출력
- id -un과 동일함

### 4. w
- 현재 로그인한 사용자가 어떤 시스템에서 로그인을 했는지, 어떤 프로세스를 실행중인지 출력함.
	- -h : 헤더 없음. no-header.
	- -s : 짧게. short.

### 5. logname
- whoami랑 동일

### 6. id
- 지정한 사용자의 사용자 정보, 그룹 정보를 출력 or 현재 사용자 정보 출력. w랑 비슷?
- 그냥 id만 쓰면 uid, gid등을 출력함. 기본적으로 숫자로 출력됨.
	- -n 옵션을 쓰면 숫자가 아닌 이름으로 출력함.

### 7. groups
- 특정 사용자나 현재 로그인한 사용자가 속한 그룹 출력. id -Gn과 동일

### 8. lslogins
- wtmp나 btmp로그를 바탕으로 시스템 전체 사용자 출력
- 정보 취득을 위한 경우 /etc/shadow랑 passwd 파일도 참조함
- -u 옵션으로 시스템 사용자를 제외시킴

### 9. pwconv, pwunconv, grpconv, grpunconv
- shadow 사용 유무를 결정함. shadow 사용시 passwd에 비번은 x표시 되어있던거 기억하지?
- 그냥 conv는 passwd를 가지고 shadow를 만듬. unconv는 shadow를 지워버리고 passwd에 암호화된 값을 집어넣음. 

### 10. pwck, grpck
- passwd랑 group 파일에 잘못된 내용 체크함
