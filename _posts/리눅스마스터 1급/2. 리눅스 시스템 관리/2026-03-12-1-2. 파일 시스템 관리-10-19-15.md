## 1. 파일 및 디렉터리 관리
### 1. 소유권과 허가권
- 이건 잘 알지
- rwx로 나뉨
- r은 읽기, w는 쓰기, x는 실행.
- 파일 유형 / 소유자 / 그룹 / 다른 사람 허가권으로 나뉨.
- 확인하려면 `ls -l` 옵션이나 `ls -n`을 사용.
- 파일 유형은 아래와 같음.
	- `-` : 일반 파일
	- d : 디렉터리
	- `|` : 심볼릭 링크
	- b : 블록 디바이스
	- c : 캐릭터 디바이스
	- p : FIFO
	- s : 소켓
- 허가권은 잘 아니까 패스
	- x가 있어야 디렉터리 안으로 접근 할 수 있다.

1. 허가권 변경은 chmod. 소유권은 chown.
	- `chmod 644 myfile.txt` 같이 사용.
	- 숫자 말고 사용자 허가 u, 그룹 g, 다른 사용자 o, 모든 사용자 a를 사용하여 `chmod u=rw myfile.txt` 이렇게 알파벳으로도 가능.
	- -R 옵션으로 디렉터리 이하의 모든 파일이나 디렉터리에도 동일한 권한 부여 가능. recursive.
	- -f : 에러 메시지 출력
	- -C : changes. 변경된 파일이나 디렉터리에 대한 정보 출력.
2. 소유권 변경은 루트만 가능함. sudo를 사용하자
	- 옵션은 허가권과 동일.
	- 그룹은 앞에 콜론을 붙여야 한다. `sudo chown :mygroup myfile.txt` 이런식. 
	- 숫자로도 지정 가능. `sudo chown 2000:2001 myfile.txt`
	- 파일 여러 개도 지정 가능함
3. 그룹의 소유권만 변경할때는 chgrp. chown은 루트만 가능했지만 얘는 본인이 소유한 파일에 대해 본인이 속한 그룹 내에서 소유권 변경 가능. 옵션은 역시나 비슷함. 
	- `chgrp colors blue.txt` : 텍스트에 대하여 그룹 소유권을 colors로 변경
4. **umask**
	- 기본 허가권 지정. 기본 허가권 값은 666. 디렉터리는 777. 만약 umask가 022라면 파일의 경우 666에서 022를 뺀 644가 되고, 디렉터리는 755가 된다.
	- 옵션 없이 입력하면 현재 값을 확인함.
	- -S 옵션으로 숫자가 아닌 문자로 확인 가능.
	- `umask u=rwx,g=rx,o=rx` 이런식. a도 사용가능, 숫자도 사용 가능.

### 2. 특수 권한
- Set-UID 권한 비트, Set-GID 권한 비트, 스티키 비트라는 특수한 권한이 존재한다.
- 이 모든건 실행 권한이 있어야 한다.
- **1. Set-UID 권한 비트**
	- 파일에 이 비트가 설정되면 사용자가 파일을 실행했을 때, 해당 사용자의 권한이 아닌 소유자의 권한으로 실행이 된다. 
	- 예를 들어 /etc/passwd에 이 권한 비트를 설정해두면 다른 사용자가 passwd를 실행할 때 임시로 루트의 권한으로 실행할 수 있음.
	- ls 명령으로 보면 소유자 허가권에 x대신 s가 표기되어 있다.
	- 만약 실행권한이 없는 파일에 설정하면 대문자로 S가 표기됨. 특수 권한이 있는데 정작 실행 권한이 없으니까 설정하라고 알려주는거임.
		- `chmod u+s hello.sh`
		- `chmod u=srwx hello.sh`
		- `chmod 4744 hello.sh`
		- 위 같은 방식으로 가능하다. 
- **2 .Set-GID 권한 비트**
	- 비슷한 느낌이다. 대신 얘는 그룹.
	- 얘는 g에다가 추가해주면 된다. 숫자로는 2.
		- `chmod g+s hello.sh`
		- `chmod g=srwx hello.sh`
		- `chmod 2744 hello.sh`

- **3. 스티키 비트**
	- 얘는 디렉터리에만 적용 가능함. 모든 사용자가 접근가능하도록 특정 디렉터리에 777을 사용한다고 치자. 근데 이러면 모두가 삭제가 가능함.
	- 스티키 비트를 디렉터리에 지정하면 다른건 다 가능하지만, 삭제는 오직 파일 소유자만 가능하다.
	- 다른 사용자 권한이 t로 설정되어있다. 얘도 만약 실행 권한이 없으면 T가 나옴.
		- `chmod o+t newdir`
		- `chmod 1777 newdir`
		- 이런 느낌이다.

- 이미 존재하는 파일에 대한 포인터를 갖는 파일 링크 기능이 있다. 뭐 잘 아는거. 그냥 하드 링크냐, 심볼릭이냐. 하드는 그냥 아예 복사, 심볼릭은 바로가기 느낌.
