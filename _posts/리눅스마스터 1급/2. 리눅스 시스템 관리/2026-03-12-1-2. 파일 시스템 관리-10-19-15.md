## 1. 파일 및 디렉터리 관리
### 1. 소유권과 허가권
- 이건 잘 알지
- rwx로 나뉨
- r은 읽기, w는 쓰기, x는 실행.
- 파일 유형 / 소유자 / 그룹 / 다른 사람 허가권으로 나뉨.
- 확인하려면 `ls -l` 옵션이나 `ls -n`을 사용.
- 파일 유형은 아래와 같음.
	- `-` : 일반 파일
	- d : 디렉터리
	- `|` : 심볼릭 링크
	- b : 블록 디바이스
	- c : 캐릭터 디바이스
	- p : FIFO
	- s : 소켓
- 허가권은 잘 아니까 패스
	- x가 있어야 디렉터리 안으로 접근 할 수 있다.

1. 허가권 변경은 chmod. 소유권은 chown.
	- `chmod 644 myfile.txt` 같이 사용.
	- 숫자 말고 사용자 허가 u, 그룹 g, 다른 사용자 o, 모든 사용자 a를 사용하여 `chmod u=rw myfile.txt` 이렇게 알파벳으로도 가능.
	- -R 옵션으로 디렉터리 이하의 모든 파일이나 디렉터리에도 동일한 권한 부여 가능. recursive.
	- -f : 에러 메시지 출력
	- -C : changes. 변경된 파일이나 디렉터리에 대한 정보 출력.
2. 소유권 변경은 루트만 가능함. sudo를 사용하자
	- 옵션은 허가권과 동일.
	- 그룹은 앞에 콜론을 붙여야 한다. `sudo chown :mygroup myfile.txt` 이런식. 
	- 숫자로도 지정 가능. `sudo chown 2000:2001 myfile.txt`
	- 파일 여러 개도 지정 가능함
3. 그룹의 소유권만 변경할때는 chgrp. chown은 루트만 가능했지만 얘는 본인이 소유한 파일에 대해 본인이 속한 그룹 내에서 소유권 변경 가능. 옵션은 역시나 비슷함. 
	- `chgrp colors blue.txt` : 텍스트에 대하여 그룹 소유권을 colors로 변경
4. **umask**
	- 기본 허가권 지정. 기본 허가권 값은 666. 디렉터리는 777. 만약 umask가 022라면 파일의 경우 666에서 022를 뺀 644가 되고, 디렉터리는 755가 된다.
	- 옵션 없이 입력하면 현재 값을 확인함.
	- -S 옵션으로 숫자가 아닌 문자로 확인 가능.
	- `umask u=rwx,g=rx,o=rx` 이런식. a도 사용가능, 숫자도 사용 가능.

### 2. 특수 권한
- Set-UID 권한 비트, Set-GID 권한 비트, 스티키 비트라는 특수한 권한이 존재한다.
- 이 모든건 실행 권한이 있어야 한다.
- **1. Set-UID 권한 비트**
	- 파일에 이 비트가 설정되면 사용자가 파일을 실행했을 때, 해당 사용자의 권한이 아닌 소유자의 권한으로 실행이 된다. 
	- 예를 들어 /etc/passwd에 이 권한 비트를 설정해두면 다른 사용자가 passwd를 실행할 때 임시로 루트의 권한으로 실행할 수 있음.
	- ls 명령으로 보면 소유자 허가권에 x대신 s가 표기되어 있다.
	- 만약 실행권한이 없는 파일에 설정하면 대문자로 S가 표기됨. 특수 권한이 있는데 정작 실행 권한이 없으니까 설정하라고 알려주는거임.
		- `chmod u+s hello.sh`
		- `chmod u=srwx hello.sh`
		- `chmod 4744 hello.sh`
		- 위 같은 방식으로 가능하다. 
- **2 .Set-GID 권한 비트**
	- 비슷한 느낌이다. 대신 얘는 그룹.
	- 얘는 g에다가 추가해주면 된다. 숫자로는 2.
		- `chmod g+s hello.sh`
		- `chmod g=srwx hello.sh`
		- `chmod 2744 hello.sh`

- **3. 스티키 비트**
	- 얘는 디렉터리에만 적용 가능함. 모든 사용자가 접근가능하도록 특정 디렉터리에 777을 사용한다고 치자. 근데 이러면 모두가 삭제가 가능함.
	- 스티키 비트를 디렉터리에 지정하면 다른건 다 가능하지만, 삭제는 오직 파일 소유자만 가능하다.
	- 다른 사용자 권한이 t로 설정되어있다. 얘도 만약 실행 권한이 없으면 T가 나옴.
		- `chmod o+t newdir`
		- `chmod 1777 newdir`
		- 이런 느낌이다.

### 3. 파일 링크
- 이미 존재하는 파일에 대한 포인터를 갖는 파일 링크 기능이 있다. 뭐 잘 아는거. 그냥 하드 링크냐, 심볼릭이냐. 하드는 가리키는 아이노드가 같고, 심볼릭은 바로가기 느낌. 
- 하드 링크는 아이노드를 가리키고 심볼릭은 경로를 가리킴.
- 심볼릭은 소프트 링크라고도 함
- 스티키 비트를 설정하지 않으면서 디렉터리를 공용으로 사용하고자 한다면 하드링크도 방법. 둘 다 동일 아이노드를 가리키기 때문에 실제 대상 파일에 접근할 수 있음. 
- 심볼릭 링크는 라이브러리 업그레이드에 편함.
- **하드 링크**
	- 동일 파일 시스템 내에서만 가능, 디렉터리 링크 불가.
	- 대상 파일의 허가권 변경시 하드 링크 허가권도 변경됨
	- ln 명령어를 통하여 하드 링크 생성 가능. `ln -lia` 명령어를 사용하여 대상과 링크 파일의 아이노드 값이 같은 것을 알 수 있음. 
		- `ln 원본파일 하드링크파일` 방식으로 사용.
- **심볼릭 링크**
	- 파일 시스템 상관없음
	- 디렉터리 가능
	- 대상 파일이랑 허가권이 다름
	- 경로만을 참조
	- ln -s로 심볼릭 링크 생성. -> 기호를 통하여 어떤 파일을 가리키는지도 알 수 있음.

### 4. 디렉터리 관리
- **pwd**
	- 현재 어떤 디렉터리에 있는지.
	- pwd는 셸 내장 명령어와 bin에 위치한 외부 명령어가 있다. 현재 사용중인 pwd가 무엇인지 알기 위해서는 `type` 명령어를 사용. `type pwd`
- **cd**
	- 체인지 디렉토리. 변경.
	- /로 시작하면 루트 디렉터리를 기점으로 절대 경로.
	- 없으면 상대 경로. 현재 내 위치를 기준으로 변경 가능.
	- ..은 상위 디렉터리로 이동하라는 뜻. 
	- .은 현재 디렉터리를 의미. 고로 cd .는 아무 변화 없음
	- ~는 현재 로그인한 계정의 홈 디렉터리를 의미. `cd ~/Documents` 는 현재 내 홈 디렉터리의 Documents 폴더로 이동하라는 뜻.
	- `-`는 이전에 방문했던 디렉터리를 나타냄.
- **mkdir**
	- 디렉터리 생성. 여러개 한거번에 가능.
		- -m : mode. 허가권 지정. chmod랑 동일하게 하면 됨.
		- p : parents. 디렉터리 생성시 상위 디렉터리가 필요하면 생성함.
	- `mkdir -p ./albums/photos` : 앨범 디렉터리 없으면 생성하고 안에 포토 디렉터리 만듬.
- **rmdir**
	- 디렉터리 삭제. 여러 디렉터리 한번에 삭제 가능.
	- `rmdir -p mydir/mydir2/mydir3` : -p로 연쇄삭제 가능. 하지만 다 비어있어야 함.
	- rm -r 옵션으로 비어있지 않은 디렉터리 강제 삭제 가능. 

### 5. 파일 관리
 - **ls**
	 - 존재하는 파일과 디렉터리 리스트 출력. 옵션 뒤지게 많음.
	 - -a : all, 숨겨진 파일도 출력
	 - -F : classify, 파일명마다 파일을 뜻하는 문자를 붙임. `@`는 심볼릭, `*`는 실행 파일, `=`는 소켓, `|`는 파이프, `/`는 디렉터리.
	 - -l : 한줄씩 자세한 정보를 출력
	 - -i : 아이노드 번호 출력
	 - -R : 목록 중 디렉터리 있으면 그 안 파일 목록도 출력
	 - -S : 파일 크기 정렬
	 - -1 : 한줄에 하나의 항목만 출력
	 - -t : 수정 시간 정렬
	 - -u : 마지막 접근 시간 출력
	 - -d : 디렉터리 자체에 대한 정보를 출력
- **cp**
	- 복사임
	- 출력 확인하려면 -v 사용
	- -r, -R : 디렉터리 내 파일이나 디렉터리도 복사
	- -i : 존재하는 파일 덮어쓰기 전 확인받기. interactive
	- -f : 덮어쓰기가 안되면 그 파일을 삭제하고 복사 시도. force.
	- -b : 대상 파일이 이미 존재하면 기존 파일명 뒤에 `~`를 붙여 백업
	- -s : 파일 복사 대신 심볼릭 링크 만듬. 단 소스 파일은 절대 경로, 대상 경로는 현재 디렉터리가 아니여야 함.
	- -p : 허가, 소유권, 접근 / 수정 시간을 보존하여 복사. 안쓰면 현재 사용자 정보 사용.
	- -v : 항목 정보를 상세히 출력
	- -a : 모든 정보를 유지하여 복사
	- -l : 하드 링크 생성
	- -u : 원본파일이 더 최신이거나 대상 파일이 없을때만 복사. 
- **rm**
	-  지정한 파일 삭제
	- 디렉터리 삭제시 -r / -R 사용.
	- 역시 i는 확인 받는 옵션. 그 외에는 f랑 r밖에 없다.
	- 만약 파일 이름에 -가 들어간다면 rm 뒤에 --을 써야 한다. 그래야 옵션 취급을 안함. `rm -- -dashfile.txt 
	- 아니면 앞에 `./`를 쓰던가.

> **shred 명령어**<br>
> - rm 명령어는 파일 시스템 상 실제 데이터를 지우지 않지만 얜 지움. 얘는 추후에 데이터를 재생할 수 없도록 다른 데이터로 덮어씀.
> - `shred -u file.txt` : 파일을 삭제하고 데이터 복구 못하도록 데이터를 여러번 덮어쓴다.
{: .prompt-info }

- **mv**
	- 이동. 얘는 i 옵션이 대문자다. -I를 써야 함.
	- 그 외에는 강제인 f, 백업인 b, 최신 경우에 덮어쓰지 않는 u, 모든 원본 파일을 지정한 경로로 이동하는 t가 있다.
- **touch**
	- 파일의 마지막 접근 시간, 수정 시간, 변경 시간과 같은 파일 스탬프 정보를 수정. 파일이 없는 경우 빈 파일을 생성.
	- -a : atime 변경. 접근 시간 변경
	- -m : mtime. 수정 시간 변경
	- -c : 파일 생성 안함
	- -r : 현재 시간이 아닌 지정한 파일의 타임 스탬프로 변경. 지정한 파일의 권한이 있어야 함.
	- -t : 현재 시간이 아닌 지정한 타임 스탬프로 변경
		- 형식은 `[[CC]YY]MMDDhhmm[.ss]`
		- 미래나 과거로 변경할 경우 쓰기 권한 + 소유자여야 가능.
	- `touch -mtime 202110251700` : 2021년 10월 25일 오후 5시 00분으로 수정 시간 변경.
- **file**
	- 파일 유형 확인.
	- -b : 파일 유형만 출력. brief의 약자
- **find**
	- 파일이나 디렉터리의 이름, 생성 날짜 등 조건에 부합하는 파일 시스템에 존재하는 파일이나 디렉터리 찾는 명령어. 아 옵션 뒤지게 많네 진짜
	- 근데 대충 보면 알 듯
	- -exec : 파일을 찾을때마다 실행할 명령어 지정
	- -ok : exec와 동일, 하지만 명령 수행 여부를 물어봄
	- -print : 표준 출력으로 개행 문자로 종료하는 전체 파일 경로 출력
		- 널로 끝나게 하려면 -print0
	- -fprint : 전체 파일 경로 출력
	- -prune : 찾은 파일이 디렉터리인 경우 하위 내용은 검색하지 않음.
	- 정규식으로 패턴 지정도 가능. 정규식은 나중에 나옴.

### 6. 텍스트 관련 명령어
- **cat**
	- 텍스트 내용을 보거나 복사하거나 합치는 기능을 수행.
	- `cat < readme.txt` 도 된다.
	- `cat readme.txt > newreadme.txt`  : 리드미를 읽어서 뉴리드미로 만듬.
	- `cat readme.txt >> newreadme.txt`  : 리드미를 읽어서 뉴리드미 뒤에다가 붙임. 만약 뉴리드미가 없으면 생성.
- **head**
	- 옵션 미지정시 첫 10줄을 출력함.
	- -n : 출력 수 지정
	- -c : 출력 바이트 지정.
- **tail**
	- head의 반대. 
	- 옵션은 -f만 추가됨. f는 새로운 데이터가 들어올 때까지 모니터링을 함.
- **more**
	- 텍스트 파일 내용이 방대해 한 페이지씩 보여주는 명령어.
	- -num :  한번에 보여줄 수를 설정.
- **less**
	- vi 명령어와 다르게 텍스트 파일을 모두 읽어 메모리에 올리지 않는다. = 시작이 빠름. 이기적 교재 만든 사람 어휘가 그냥 딸림. 이과는 글쓰지 말라고
	- -c : clean. 화면을 지우고 최상단부터 출력.
	- -s : squeeze. 연속된 빈 줄을 한 줄로 만듬.
	- -N : 행 번호랑 같이 출력
	- -e : 파일 끝에서 다시 끝으로 이동하려하면 종료됨.
	- `+ 행번호`를 사용하여 특정 줄부터 출력 가능
- **grep**
	- 텍스트 파일을 한 줄씩 읽어 지정한 패턴과 일치하는 문자열을 보여줌.
	- `grep 옵션 패턴 파일` 식으로 사용함.
	- `grep francis /etc/passwd` : passwd에서 francis라는 문자열을 찾음.
	- 옵션 존나 많네 아 이 개같은 옵션지옥
	- `grep -r francis /www/` 같이 디렉터리 이하의 모든 파일에서 찾는 것도 가능함.
	- -w : 정확히 일치하는 단어일 경우 출력. 예를 들면 francis를 찾고자 하는데, francisco라는 단어가 있어도 정확하게 일치하지 않으므로 (부분 일치) 출력하지 않음.
	- -c : 일치하는 줄 개수 출력
	- -r : 모든 파일에 대하여 패턴 찾기
	- -o : 일치하는 부분만 출력
	- 걍 다른거 나오면 틀릴게

> **정규표현식**<br>
> - . : 하나의 문자와 매칭
> - a* : a가 없거나, 그 이상이 포함되어있음.
> - `[abc]` : a,b,c 중 하나라도 포함되면 매칭
> - `[^abc]` : 하나라도 포함 안하면 매칭.
> - `[a-zA-Z]` : a~z or A-Z만 매칭.
> - `^패턴` : 패턴으로 줄이 시작할 경우. 부분 일치.
> - `패턴$` : 패턴으로 줄이 끝날 경우. 부분 일치.
> - `\<패턴` : 패턴으로 단어가 시작할 경우. 전부 일치.
> - `패턴\>` : 패턴으로 단어가 끝날 경우.
> - 여기서 역슬래시 = 원화 표시와 같음...

- **wc**
	- 지정한 파일에 대해 단어, 개행문자, 문자의 개수를 셀 수 있음.
	- 옵션 안 넣으면 줄, 단어, 문자를 알려줌.
	- -l : 라인. 
	- -w : 워드. 단어.
	- -c : 바이트.
	- -L: 가장 긴 줄 길이.
- **sort**
	- 텍스트 파일을 한 줄씩 읽어서 정렬함.
	- -r : 정렬 결과를 반대로 출력
	- -o : 특정 파일에 정렬 결과를 저장함.
	- -k : 특정 필드를 기준으로 정렬함.
	- ex) `sort -k 3.3 person.txt` : 3번째 필드 3번째 글자를 이후를 기준으로 정렬함.
- **cut**
	- 텍스트 파일을 한 줄 또는 여러 줄을 잘라낼 수 있는 명령어. 
	- -f : 필드 선택.
	- -d : 필드 구분자 변경. 기본은 탭임.
	- -c : 각 줄의 문자를 선택.
	- `cut -f 1-2 scores.txt` : 1, 2번째 필드를 출력
	- `cut -c 1-3 scores.txt` : 각 줄에서 3개 글자만 출력함.
- **split**
	- 파일을 고정 크기의 여러 파일로 분할함.
	- -b : 저장할 크기를 지정함. bytes.
	- -C : 행의 크기를 주어진 값에 맞춰서 씀.
	- -l : 파일당 줄 수를 지정.
	- `split -b 1024 /etc/passwd newpasswd`
