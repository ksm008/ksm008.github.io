## 1. 리눅스 커널
### 1. 커널의 개요
- 리누스 토발즈가 개발했다는 건 이제 보기도 싫음. GNU CPL v2 라이선스 하에 무료 공개.
- 모노리딕 커널이다. 현대 운영체제가 가져야 할 모든 특징을 가짐.
- uname -r 버전을 통해 커널 버전 확인 가능.
- 가장 앞 4는 주 버전, 18은 부버전. 뭐 대충 빌드 뭐시기 뭐시기 안중요한거

```bash
[linux@localhost ~]$ uname -r
4.18.0-477.10.1.el8_8.x86_64
```

### 2. 커널 컴파일 순서
1. 소스코드 다운로드
	- wget을 통해 다운받으면 됨
	- 그리고 tar xvzf로 압축을 해제하고
2. 커널 컴파일 필수 도구 설치
	- yum으로 컴파일에 필요한 컴파일러 및 설치 도구를 설치한다.
	- `sudo yum install -y ncureses-devel make gcc bc bison flex elfutils-libelf-devel openssl-devel grub2`
	- 뒤지게 많다.
3. 커널 환경 설정
	- make mrproper로 기존 백업파일 모두 제거. 만약 첫 설치라면 불필요
	- cp /boot/config-$(uname -r) .config로 기존 커널의 설정 파일을 복사해서 수정하기.
	- make menuconfig을 통해 원하는 커널의 기능을 활성화/비활성화.
4. 커널 컴파일
	- 대표적으로 zImage랑 bzImage가 있다. zImage는 1메가 이하의 작은 커널일때 유용하고 넘어가면 bz를 쓴다.
	- 만약 zImage를 썼는데 너무 크다는 오류가 나오면 많은 부분을 모듈화시켜 크기를 줄이거나 bz를 써라. 특히 bz는 쓰고나서 bzip2로 압축해야된다.
	- make bzimage 사용.
	- make modules : 커널에서 사용할 모듈 컴파일
	- sudo make install : 컴파일이 완료된 모듈을 /lib/modules/kernel-version 이하에 설치
	- sudo make modules_install : 커널 이미지 복사 및 부트로더 환경설정 등 커널을 시스템에 설치한다.
	- sudo reboot : 시스템을 재부팅한다.

### 3. 그 외 커널 컴파일 명령
1. 클린 타겟
	- 이전에 커널을 컴파일 했을 때 생긴 파일들은 새로 컴파일을 할 때 영향을 줄 수 있다. 이를 위해 make에 3가지 옵션을 제공함.
		- make clean : 커널 환경설정을 제외한 대부분의 파일을 제거
		- make mrproper : 포함하여 모든 파일을 모두 제거
		- make distclean : 윗 동작을 모두 수행 + 추가로 백업, 패치 파일도 모두 제거.
2. 커널 환경 설정
	- 텍스트 기반, 그래픽 기반 설정까지 다양함.
	- 환경설정이 해당 항목을 포함하려면 y, 미포함은 n, 모듈로 포함하려면 m이다.
		- make config : 텍스트 기반 환경설정 도구
		- make menuconfig : 텍스트 기반 컬러 메뉴 목록
		- make nconfig : 향상된 컬러 기반, 펑션키, 커서 사용 가능
		- make xconfig : x윈도우. Qt 4.8 이상 필요
		- make gconfig : x윈도우. GTK+ 2.0 이상 필요

## 2. 모듈
### 1. 모듈의 개요
1. 정의
	- 커널의 기능을 확장하기 위해 메모리에 동적으로 로드 가능한 커널 오브젝트. 확장자는 .ko
	- 시스템 중단 없이 동적 로드, 언로드가 가능함. 리눅스에서는 LKM (로더블 커널 모듈)이라고 부름
2. 필요성?
	- 모든 기능이 커널에 포함되어 있다면 사용 여부랑은 상관없이 항상 시스템 메모리를 차지하고 낭비가 발생함. 또한 새로운 기능 추가시 커널 소스코드를 수정, 컴파일, 교체 작업을 해야한다.
	- 하지만 모듈을 사용하면 모듈을 메모리에 로드, 제거함으로 동적으로 시스템 중단없이 사용이 가능하다.
3. 특징
	- 재시작 없이 모듈을 로드, 필요가 없어지면 삭제가 가능.
	- 커널의 일부분이여서 시스템 제약이 없다. 그래서 만약 얘가 병신이 되면 시스템이 패닉(panic)에 빠짐. 패닉에 빠지면 운영체제를 중단시키고 로그로 저장한 후 사용자에게 오류를 보여줌. 윈도우의 블루 스크린에 해당한다.
	- 시스템 모듈은 일반적으로 **/lib/modules/(커널 버전 문자열)/kernel**에 위치함.
	- `ls /lib/modules/$(uname -r)/kernel` 로 커널의 모듈을 확인 가능.

### 2. 모듈 관련 명령어
- lsmod
	- 모듈의 리스트와 정보를 출력함. /proc/moduels 파일의 내용을 기반으로 좀 더 보기좋게 출력됨.
- insmod
	- 커널에서 모듈을 로드함. 경로를 지정하지 않으면 일반경로에서 찾아서 로드함. sudo 필요!
	- 일반적으로 이거보다는 모듈의 의존성을 고려하는 **modprobe**를 주로 사용함. modprobe도 내부적으로 insmod를 사용하여 구현했기 때문.
	- 모듈을 로드할 때 옵션을 넘겨줄 수 있음. 
	- ex)`sudo insmod /home/francis/Desktop/dummy.ko mode="standlone"` 이런식.
- rmmod
	- 뭐겠어용. 얘도 sudo 필요
	- modprobe -r를 통한 언로드를 선호함.
- **modprobe**
	- 의존성을 고려하여 로드, 언로드를 함. 얘도 sudo.
	- 의존성은 **depmod**에 의해 생성된 **modules.dep** 파일을 참조함.
	- 얘도 로드시 옵션을 넘기면서 로드가 가능함.
	- 옵션은 아래와 같음.
		- -l : 리스트 출력
		- -r : 언로드할때 씀
		- -c : 모듈 관련 환경설정의 파일 내용을 출력한다.
