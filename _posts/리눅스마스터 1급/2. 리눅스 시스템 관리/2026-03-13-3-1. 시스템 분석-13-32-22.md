## 1. 시스템 로그 개요 및 분석
### 1. 개요
- **설명**
	- 대부분의 로그는 /var/log 디렉터리 아래에 저장됨
	- syslogd나 rsyslogd 같은 시스템 데몬에 의해 로그파일이 생성됨.
	- 레드햇 7 이후에서는 systemd-journald와 rsyslogd 데몬에 의해 로그가 관리된다.
		- systemd 로그는 리부팅하면 사라짐. 부팅 시점부터 각 유닛에서 발생한 모든 로그를 바이너리 타입으로 저장
		- rsyslogd는 수집된 로그를 설정된 규칙에 따라 각 로그 파일에 저장하는 역할을 수행.
- **로그 확인하기**
	- /var/log 디렉터리로 이동, ls 명령어로 로그 파일 확인 가능.
	- 레드햇 계열 대표적 시스템 로그 파일은 **/var/log/message**, 데비안은 **/var/log/syslog**이다.
	- less 명령어로 로그 파일을 확인하려면 루트 권한이 필요.

### 2. 주요 시스템 로그 파일
- 다음과 같다.
	- /var/log/message : 전체적인 로그
	- /var/log/secure : 로그인 행위 기록
	- /var/log/boot.log
	- /var/log/maillog
	- /var/log/kern
	- /var/log/dmesg : 디바이스 메시지
	- /var/log/faillog
	- /var/log/cron
	- /var/log/yum.log
	- /var/log/httpd/
	- /var/log/mysqld.log
	- /var/log/xferlog : ftp
	- /var/log/lastlog : 각 사용자의 마지막 로그인 기록
	- /var/log/wtmp : 각 사용자의 매 로그인, 로그아웃 기록
	- /var/log/btmp : 모든 로그인 실패 기록
	- /var/log/utmp : 현재 로그인 상태 보관

### 3. 시스템 로그 파일 명령어
- **dmesg**
	- 커널 링 버퍼를 제어하거나 출력하는 명령어.
	- **옵션**
		- -C : 내용을 출력 후 지운다. clear
		- -ㅣ : 주어진 레벨만 출력한다.
			- alert, crit, err, warn, sgfault 레벨이 존재.
		- -T : 사람이 읽을 수 있는 날짜 형식으로 타임 스탬프를 출력
- **lastlog**
	- /var/log/lastlog 파일을 출력함
	- **옵션**
		- -b : 지정한 날짜보다 이전인것 출력
		- -t : 지정한 날짜보다 이후인것
		- -u : 지정한 사용자 로그만 출력
- **last**
	- /var/log/wtmp 파일 출력. 
	- **옵션**
		- -n : 몇 줄 출력할건지
		- -R : 호스트명 출력하지 않음
		- -a : 마지막 컬럼의 호스트명 필드 추가
		- -d : 외부 로그인에 한하여..
		- -x : 시스템의 셧다운 로그와 실행 레벨 변화를 출력
		- -f file : 지정한 파일을 보여줌
		- -t 시간 : 지정한 시간의 로그인 기록 출력
		- -F : 모든 정보 출력
		- -i : 호스트명 대신 아이피 주소를 출력
		- -w : 사용자 전체 이름과 도메인 이름을 출력
- **lastb**
	- 로그인 실패 기록을 출력함. /var/log/btmp를 기준.
	- 옵션은 last랑 같음.

## 2. 시스템 로그 관리
### 1. 시스템 로그 관리
- **생성 및 관리**
	- rsyslog가 더 인기 많음
- **syslog**
	- 에릭 알만이 개발함
	- /etc/syslog.conf 설정 파일 기반, /var/log 디렉터리에 로그를 생성
- **rsyslog**
	- 레이너 게르하드가 주축으로 개발 시작
	- IP 통신을 통한 로그 기능 구현을 목적
	- **/sbin/rsyslogd** 데몬을 통해 로그 기능을 수행.
	- **/etc/rc.d/init.d/syslogd** 스크립트를 통해 부팅시 데몬이 시작됨
	- **/etc/rsyslog.conf** 설정 파일 기반 /var/log 디렉터리에 로그 생성.
	- 멀티 쓰레드 지원, 다양한 프로토콜 지원, 다양한 데이터베이스 지원 등등 하는것도 많아요

### **2. 로그 관리**
- **설정 및 시작**
	- `sudo rpm -qv rsyslog` 로 확인하고 없으면 yum으로 설치
	- **/etc/rsyslog.conf** 파일을 통해 로그 생성 규칙을 설정함.
		- `$template FILENAME,"템플릿"`을 통해 파일 생성 규칙 설정.
		- 막줄에는 ?FILENAME
- **rsyslog 관련 파일**
	- /etc/rc.d/init.d/rsyslog 
		- 시스템이 시작될때 데몬을 실행하는 스크립트
		- stop, start, restart 사용 가능
	- /etc/rsyslog.conf
		- 데몬의 환경설정 파일
	- /etc/sysconfig/rsyslog
		- rsyslogd 데몬을 실행할 때 옵션을 설정함
	- /sbin/rsyslogd 
		- rsyslogd의 데몬 파일 경로
- **/etc/rsyslog.conf 파일**
	- Global directives
		- rsyslog 데몬 전역 설정
		- 메세지 큐 크기 설정, 외부 모듈 로드 등이 가능
	- Templates
		- 로그 메세지 형식을 템플릿으로 보관
		- Rules 섹션처럼 이 템플릿 사용 가능
	- Output channel 
		- 템플릿처럼 출력 채널에 대한 세부 설정을 저장해두고 뭐시기 뭐시기
	- **Rules(selector + action)**
		- 로그 출력 규칙을 설정한다
	- Rules 섹션의 각 Rule은 selector와 action으로 구성되고, selector는 다시 facility와 priority로 구성된다.
	- 딥하게는 안나오는듯..?

### 3. 로테이션을 통한 로그 용량 관리
- **logroate 설명**
	- 로그 파일로 인해 시스템 저장소가 꽉 차는 것을 막고 공간을 효율적으로 쓰기 위해 설계한 시스템 관리자를 위한 유틸리티.
	- **cron**에 의한 하루에 한 번 등록된 스크립트를 실행.
	- **/etc/logrotate.conf** 환경 설정을 읽어서 **/usr/sbin/logrotate** 명령을 실행함.
	- 기본 구성은 다음과 같다.
	- 매일/주/달/년 단위로 로테이트 설정 가능. 로그파일이 없으면 넘어가는 옵션은 missingok
	- 최대 로테이트 횟수 설정
	- 특정 로그에 대해서도 설정이 가능함.
	- 로테이션 이력은 **/var/lib/logrotate.status**에서 확인이 가능하다.

### 4. journalctl을 통한 로그 관리
- systemd에서 제공하는 커널 및 저널 로그를 관리하기 위한 명령어
- `journalctl -r` : 역순 출력
- -n 숫자 : 지정한 수만큼 로그를 출력
- -p 레벨 : 지정한 우선순위 이상의 모든 로그를 출력
- --list-boots : 시스템 부팅시의 모든 로그 출력
- -since=today : 특정 날짜 이후 로그 출력. 타임스탬프도 가능
- -since= --until= : 기간 설정하여 로그 출력
