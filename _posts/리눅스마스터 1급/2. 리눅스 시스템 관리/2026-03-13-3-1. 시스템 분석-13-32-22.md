## 1. 시스템 로그 개요 및 분석
### 1. 개요
- **설명**
	- 대부분의 로그는 /var/log 디렉터리 아래에 저장됨
	- syslogd나 rsyslogd 같은 시스템 데몬에 의해 로그파일이 생성됨.
	- 레드햇 7 이후에서는 systemd-journald와 rsyslogd 데몬에 의해 로그가 관리된다.
		- systemd 로그는 리부팅하면 사라짐. 부팅 시점부터 각 유닛에서 발생한 모든 로그를 바이너리 타입으로 저장
		- rsyslogd는 수집된 로그를 설정된 규칙에 따라 각 로그 파일에 저장하는 역할을 수행.
- **로그 확인하기**
	- /var/log 디렉터리로 이동, ls 명령어로 로그 파일 확인 가능.
	- 레드햇 계열 대표적 시스템 로그 파일은 **/var/log/message**, 데비안은 **/var/log/syslog**이다.
	- less 명령어로 로그 파일을 확인하려면 루트 권한이 필요.

### 2. 주요 시스템 로그 파일
- 다음과 같다.
	- /var/log/message : 전체적인 로그
	- /var/log/secure : 로그인 행위 기록
	- /var/log/boot.log
	- /var/log/maillog
	- /var/log/kern
	- /var/log/dmesg : 디바이스 메시지
	- /var/log/faillog
	- /var/log/cron
	- /var/log/yum.log
	- /var/log/httpd/
	- /var/log/mysqld.log
	- /var/log/xferlog : ftp
	- /var/log/lastlog : 각 사용자의 마지막 로그인 기록
	- /var/log/wtmp : 각 사용자의 매 로그인, 로그아웃 기록
	- /var/log/btmp : 모든 로그인 실패 기록
	- /var/log/utmp : 현재 로그인 상태 보관

### 3. 시스템 로그 파일 명령어
- **dmesg**
	- 커널 링 버퍼를 제어하거나 출력하는 명령어.
	- **옵션**
		- -C : 내용을 출력 후 지운다. clear
		- -ㅣ : 주어진 레벨만 출력한다.
			- alert, crit, err, warn, sgfault 레벨이 존재.
		- -T : 사람이 읽을 수 있는 날짜 형식으로 타임 스탬프를 출력
- **lastlog**
	- /var/log/lastlog 파일을 출력함
	- **옵션**
		- -b : 지정한 날짜보다 이전인것 출력
		- -t : 지정한 날짜보다 이후인것
		- -u : 지정한 사용자 로그만 출력
- **last**
	- /var/log/wtmp 파일 출력. 
	- **옵션**
		- -n : 몇 줄 출력할건지
		- -R : 호스트명 출력하지 않음
		- -a : 마지막 컬럼의 호스트명 필드 추가
		- -d : 외부 로그인에 한하여..
		- -x : 시스템의 셧다운 로그와 실행 레벨 변화를 출력
		- -f file : 지정한 파일을 보여줌
		- -t 시간 : 지정한 시간의 로그인 기록 출력
		- -F : 모든 정보 출력
		- -i : 호스트명 대신 아이피 주소를 출력
		- -w : 사용자 전체 이름과 도메인 이름을 출력
- **lastb**
	- 로그인 실패 기록을 출력함. /var/log/btmp를 기준.
	- 옵션은 last랑 같음.

## 2. 시스템 로그 관리
### 1. 시스템 로그 관리
- **생성 및 관리**
	- rsyslog가 더 인기 많음
- **syslog**
	- 에릭 알만이 개발함
	- /etc/syslog.conf 설정 파일 기반, /var/log 디렉터리에 로그를 생성
- **rsyslog**
	- 레이너 게르하드가 주축으로 개발 시작
	- IP 통신을 통한 로그 기능 구현을 목적
	- **/sbin/rsyslogd** 데몬을 통해 로그 기능을 수행.
	- **/etc/rc.d/init.d/syslogd** 스크립트를 통해 부팅시 데몬이 시작됨
	- **/etc/rsyslog.conf** 설정 파일 기반 /var/log 디렉터리에 로그 생성.
	- 멀티 쓰레드 지원, 다양한 프로토콜 지원, 다양한 데이터베이스 지원 등등 하는것도 많아요

### **2. 로그 관리**
- **설정 및 시작**
	- `sudo rpm -qv rsyslog` 로 확인하고 없으면 yum으로 설치
	- /etc/rsyslog.conf 파일을 통해 로그 생성 규칙을 설정함.
