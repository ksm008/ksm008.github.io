## 1. 백업의 개요
### 1. 필요성
- 비즈니스 연속성
- 데이터의 중요성
- 시스템 장애 발생 위험

### 2. 전략의 요건
- 복구가 가능해야 함
- 장애 발생 이전으로 복구 가능해야함
- 최소 손실, 노력, 비용 가성비충
- 초기 백업 이후 최대한 자동화가 되어야 하는 날먹충

### 3. 백업 전략 수립 시 고려사항
- 무엇을, 언제, 어디에 백업할 것인가?
- **적용할 백업 유형**
	- 전체 백업 : 전체를 싹 다 함. 간단하지만 용량이 크겠죠
	- 증분(incremental) 백업 : 전체 백업 이후, 변경된 데이터만 백업함. 결국 복구하려면 전체와 변경된 데이터 모두 있어야 복원이 가능
	- 차등(differential) 백업 : 지난번 백업에 이번 변경 데이터를 누적하여 데이터를 생성. 최근 백업 데이터만으로도 전체 복원 가능.
- 압축과 암호화를 할 건가?
- 백업 검증은 어떻게?
- 유틸리티와 서비스는 뭘 쓸거임?
시어머니 납셨네 그냥

## 2. 파일 백업
### 1. tar
- 아카이브를 생성하거나 추출할 때 사용. 디스크 공간을 적게 차지함.
- **전체 백업**
	- `tar -cvfp archive.tar /path/to/dir`
		- -c는 아카이브 생성, -v는 상세정보 출력, -f는 아카이브 파일명 지정, -p는 접근 권한 유지.
		- -v랑 -p는 백업을 위한 필수 옵션은 아님
	- `tar -xvf archive.tar`
		- -x는 아카이브를 추출함.
	- tar로 백업시 -z나 -j 옵션을 사용하여 gz, bz2 형식으로 압축 가능.
		- -z는 gz, j는 bz2. 
		- 복원할때도 마찬가지.
		- `tar -xvjf, tar -cvzf` 이런식.
- **증분 백업**
	- tar -g 옵션을 사용한다.
	- 스냅샷으로 보관함.
	- 이전 백업 파일이 없는 최초 백업 파일을 레벨 0 백업이라고 함.
		- `tar -g snapshot -cvfp archive-level0.tar /path/to/dir`
	- 레벨 1 백업시에는 기존 레벨 0 백업의 스냅샷 파일을 통해 변경된 사항만 level1에 백업함.
		- `tar -g snapshot -cvfp archive-level1.tar /path/to/dir`
	- 이후 복원할때는 백업을 한 순서대로 복원.

### 2. cpio
- 아카이브로 파일을 복사하거나 추출할 때 사용.
- cpio 포맷 뿐 아니라 tar 파일 형식도 지원.
	- 바이너리, ASCII, HP-UX, tar 형식을 지원한다.
- tar는 많은 파일을 하나로 아카이빙 목적, cpio는 하나의 디렉터리에서 다른 디렉터리로 파일을 복사하기 위한 목적
- **파일 시스템에 직접 접근을 못하고 표준 입력으로 받기 때문에 증분 백업이 불가능함**
- Copy-out, in, pass 총 3가지 모드가 있다. 백업시에는 인, 아웃이 사용되고 복사시에는 패스가 사용된다.
- **주요 옵션**
	- -o : 아웃모드
	- -i : 인모드
	- -p : 패스모드
	- -v : 처리중인 파일 목록 출력
	- -c : SVR4 형식 사용.
	- -d : 디렉터리 생성
	- -t : 입력으로 들어오는 목록을 출력
	- -F : 표준 입/출력을 사용하지 않고 아카이브할 파일을 지정할 때 사용
	- -H : 아카이브 포맷 지정
	- -0 : null로 종료되는 파일 목록을 읽음
- ex)
	- `ls | cpio -o tar > home/test-account/backup.tar`
	- `cpio -iv < home/test-account.backup.cpio`

## 3. 파일 시스템 및 디스크 백업
### 1. dump
- 파티션 단위로 백업할 때 사용하는 명령어
	- -0 ~ 9 : 레벨을 지정. 0단계는 전체, 1단계부터는 부분 백업.
	- -f : 파일이나 장치 지정.
	- -u : 덤프 완료 후 /etc/dumpdates라는 파일에 백업 정보를 기록한다.
- `dump -0u -f backup.dump /dev/hda7` : /dev/hda7을 백업하여 backup.dump를 생성함.
- `dump -0f /dev/nst/0 /home` : 홈 디렉터리 전체를 /dev/nst/0 장치에 백업

### 2. dd
- 지정한 장치를 지정한 형식으로 변환하는 명령어.
- 파티션이나 디스크 단위로 백업할 때 쓰거나
- CD장치에서 ISO 이미지 추출시에 씀
- **옵션**
	- if : 표준 입력 대신 지정한 파일이나 장치에서 읽음
	- of : 출력 대신 파일이나 장치로 씀
	- bs : 블록을 읽거나 쓸 때 크기를 지정. 바이트 단위
- `dd if=/dev/sda1 of=/dev/sdb bs=1k` : /dev/sda1 장치를 /dev/sdb 장치로 백업하고 블록 크기는 1KB. 백업시 disk.img 같은 파일로도 가능.

### 3. restore
- dump 명령으로 생성한 백업 파일 복원할때 씀
	- -i : 대화 모드를 통한 복원 파일 지정
	- -f : 복원 파일 지정. 테이프나 디스크 같은 장치일수도 있음.
	- -r : 파일 시스템을 복원함. 파일 시스템은 마운트가 되어있어야 하고 보통은 증분 백업 이미지 복원함.
- `restore -rf backup.dump`

## 4. 네트워크 백업
### 1. rsync
- 네트워크로 연결된 리모트 컴퓨터의 파일들을 동기화하는 도구. 속도가 빠름
- 로컬 컴퓨터와 다른 원격 호스트, 원격 셸, rsync 데몬 간 파일 복사가 가능함. 다양한 옵션을 제공하고 유연한 방식을 제공함.
- 델타 전송 알고리즘을 적용해 네트워크 전송 속도를 향상시킴. 
- 백업이나 미러링 등의 작업에서 사용됨
- **옵션**
	- -r : 하위 디렉터리까지 복사
	- -l : 파일 링크를 그대로 보존하여 복사
	- -p : 권한을 보존하여 복사
	- -t : 파일 수정 시간을 보존
	- -g : 그룹 소유권 보존
	- -o : 소유자를 보존, 루트만 가능
	- -D : 블록 디바이스 등 특수파일로 복사 가능하게 함, 루트만 가능
	- -H : 하드 링크 보존
	- -a : 여러 파일을 하나로 묶는 아카이브 모드.
	- -v : 과정 상세 출력
	- -z : 전송할 때 압축.
	- -e : 원격지와 연결할 프로그램을 지정. 기본은 ssh
- ex) `rsync -avz /home francis:public_html/` : 홈의 모든 파일을 속성까지 동일하게 원격지 호스트의 public_html에 복사하고, 전송하는 사이에 압축함.
