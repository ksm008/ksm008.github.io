## 1. 디스크 확장
### 1. 디스크 확장의 개요
- 하드를 추가 장착하는거잖아
- 과정은 다음과 같다.
	- 하드를 부착한다
	- 확장 파티션을 생성하고
	- 논리 파티션 생성
	- 파티션 포맷
	- 마운트
	- 확인
	- /etc/fstab 등록
- 쉽죠?

### 2. 하드디스크 추가하기
1. 시스템에 부착하기
	- 하드를 꽂고 `fdisk -l` 옵션을 통해 하드 장착 여부를 확인한다.
	- 기존에 없던 장치가 추가된 것을 확인.
2. 확장 파티션 생성
	- 이제 파티션을 생성해야 한다. 
	- `sudo fdisk 인식된디스크`를 통해 기본 설명을 확인 가능. fdisk 이후 어떤 명령어를 써야 할지 모를대는 m 옵션을 사용.
	- fdisk 진입 후 n을 통해 새로운 파티션 생성 시작
	- 주 파티션이냐, 확장이냐로 구분된다. p는 주 파티션, e는 확장. 부팅 여부는 신경쓰지 않을 것이므로 주로 e를 사용함.
	- 파티션 번호, 첫 실린더 번호, 마지막 실린더 번호는 그냥 기본값을 선택한다. (default : 1 이런식으로 기본값을 알려줌)
	- 그 이후 w를 눌러 파티션 설정을 디스크에 적용한다. write의 약자인듯
	- 그리고 다시 `fdisk -l`를 써보면 새로운 확장 파티션이 생성된다.
3. 논리 파티션 생성
	- 파티션은 논리적으로 여러 개의 파티션으로 나눌 수 있음. 
	- 다시 fdisk 인식 디스크를 통해 진입함
	- 이후 n를 통해 새로운 파티션 생성
	- 그리고 1 옵션을 통해 논리 파티션 생성
	- 여러 논리 파티션 생성을 할 때는 첫 실린더, 마지막 실린더 값을 적절히 지정해야 하지만 책에선 한 개만 만드므로 기본값을 설정했음.
	- 그리고 똑같이 w로 디스크에 적용, 논리 파티션 여부 확인.
	- 추가로 **/proc/partitons** 파일에서 시스템의 파티션 정보를 확인할 수 있다.
4. 파티션 포맷
	- `sudo mkfs.ext4 /dev/sdb5` : 방금 만든 논리파티션 /dev/sdb5를 포맷함.
5. 마운트
	- 포맷을 마친 위 예시 장치를 특정 경로로 마운트해야한다. 디렉터리는 미리 생성되어있어야 하고, -t 옵션을 사용하여 마운트 대상의 파일 시스템을 지정해야 한다. ext3,4 등
	- `sudo mount -t ext4 /dev/sdb5 /home/francis/tmp` 이런 느낌.
6. 확인
	- `df -h` 명령어를 통해 마운트 여부를 확인하면 된다. 크기, 사용 가능한 용량도 나옴.
7. /etc/fstab 파일 등록
	- 리부트 후에도 자동 마운드 될 수 있도록 파일의 끝에 정보를 입력한다.
	- `[추가된 파티션] [경로] [파일 시스템] [모름] [몰라뭐임이게]`

## 2. 프린터
### 1. 리눅스 프린팅 시스템의 개요
- 초기 프리터는 라인 프린터를 썼대요 - 고정 크기, 단일 폰트를 가지는 한 줄 출력 방식. 다들 보면 아는거
- 속도가 느려서 스풀링 방식을 채택했고 지금도 씀
- 라인 프린트를 지원하기 위해 처음에 BSD 프린팅 시스템을 이용했음. 그 구성은 서버에서 동작하는 라인 프린터 데몬과 프린트 요청을 위한 lpr 명령어를 포함하는 클라이언트.
	- 당시 프로토콜은 LPDP (라인 프린터 데몬 프로토콜), LETF의 RFC1179로 표준화가 됨
- System V도 BSD 프린팅과 기능은 동일하지만 명령어가 다른 별도 프린팅 시스템을 가졌음. BSD에서는 lpr, System V는 lp 이런 식.
- 그 후 다양한 폰트 지원이나 여러가지 출력 기술이 개선되면서 기존의 프린팅 시스템을 호환하면서 LPRng (LPR 넥스트 제네레이션)과 CUPS (커먼 유닉스 프린팅 시스템)와 같은 프린팅 시스템이 개발됨.
- 결국 CUPS가 현재 리눅스, 유닉스 호환 시스템의 프린팅 표준 시스템이 되었다.

### 2. CUPS
1. 개요
	- 몰라도 됨. 대충 얘가 대표라는 뜻. 애플이 만들었다는것만?
2. 특징
	- 여러 운영체제 다 지원함.
	- HTTP 기반 IPP 프로토콜 사용, SMB 프로토콜도 지원
	- BSD와 System V 계열 명령어 지원
	- 웹 서버의 Common Log Format 형태와 로그 파일을 제공. 웹상에서도 lpadmin 명령을 통해 관리자 수행이 가능함.
3. 구성요소
	- **스케줄러**
		- 인쇄 요청을 처리하는 HTTP/1.1 서버 데몬 (cupsd)
		- 인쇄 뿐만 아니라 프린트 상태나 관리도구 역할도 함
		- 대충 뭐 이거저거 많이 하면 얘임
	- **환경설정 파일**
		- 아파치의 httpd.conf과 같이 환경설정 파일이 존재함.
		- 사용 가능한 프린터와 대기열 목록 설정, 출력을 위한 형식 변환 규칙 정의.
		- 프린터 기능을 설정하는 PPD 파일을 정의함.
			- cupds.conf : 데몬 설정파일
			- printers.conf : 각 프린터의 정보 설정
			- classes.conf : 각 프린터 설정 정보 설정
	- CUPS API
		- API임. GNU LGPL을 따름.
	- BSD, System V 명렁어
		- 호환된다구요
	- **필터**
		- 인쇄물을 프린터가 이해할 수 있는 형식으로 변환하는 프로그램.
		- 예를 들어 인쇄를 위한 문서를 PostScript 형식으로 변환하여 프린터에 요청해도 모든 프린터가 다 그걸 지원하지 않기 대문에 PostScript Raster 같은 필터로 변환을 해야함.
		- 여러가지 파일, 이미지 형식을 지원하기 때문에 필터 여러개를 조합하여 출력 포맷을 만드는 것도 가능
	- 백엔드
		- 장치나 네트워크 상의 컴퓨터로 프린트 데이터를 보내는 특별한 필터.
		- 삼바 프로토콜 지원함
		- 병렬, 직렬, 뭐 등 다양한 디바이스를 지원한다

### 4. 프린트 출력하기
- **lpr** (BSD)
	- 프린터에 작업을 요청하고 파일을 읽어서 출력 가능 데이터를 LPD에 전달함. 파일을 지정하면 그걸 요청, 아니라면 표준 입력으로 데이터를 받아 LPD에 전달한다.
	- 옵션은 아래와 같음
		- -# : 인쇄 매수를 지정
		- -m : 작업이 완료되면 이메일 알림
		- -P : 프린터를 지정
		- -C , -T, -J : 작업 이름과 제목을 설정
		- -r : 출력할 파일을 지정, 출력 완료 후 파일은 삭제가 됨
		- -l : 필터를 안쓰는 출력. -o raw 옵션과 동일하대용
	- **예시**
		- `lpr myfile.txt`
		- `cat myfile.txt | lpr`
		- `cat myfile.txt > /dev/lp0`
		- `lpr -P HL-2280DW -J "francis's text file" -#2 -m -r printexample.txt`
			- 프린터 지정, 작업 제목, 매수, 이메일 알림, 파일 지정 및 삭제 옵션을 담음.
- **lp** (System V)
	- lpr과 유사함
	- 옵션은 아래와 같음
		- -d : 프린터를 지정
		- -n : 매수 지정

### 5. 프린트 취소
- **lprm** (BSD)
	- 프린트 대기열에 있는 작업 번호를 통해 작업 취소. 번호가 없으면 현재 작업을 취소함.
	- 옵션들은 아래와 같음
		- `-` : 모든 작업을 취소함
		- -U : 특정 사용자의 작업 취소
		- -P : 특정 프린터 작업 취소
		- `-h server[:포트]` : 작업 취소할 프린터 서버를 지정
	- 예시
		- `lprm -` : 모든 작업 취소
		- `lprm 5` : 5번 작업 취소
		- `lprm -P mtdest 10` : mydest 프린터의 10번 작업을 취소
- **cancel** (System V)
	- 얘도 번호 지정 안하면 출력 중인걸 멈춤
		- -a : 모든 작업 취소
		- -U : 서버 연결 시 사용자 지정
		- -h hostname:포트 : 취소할 서버 지정
		- -u : 특정 사용자의 인쇄 작업을 취소함
	- 예시
		- `cancel -a` : 모두 취소
		- `cancel printer-7` : 작업 아이디가 printer-7인걸 취소.

### 6. 프린트 작업 및 큐 관리
- lpc (BSD)
	- 프린터와 CUPS 프린팅 시스템에서 제공하는 프린터 클래스 대기열을 제어한대요
	- 옵션
		- status
		- help, ?
		- quie, exit
		- 설명이 필요한가...
	- `/use/sbin/lpc status HL-2280DW`
- lpq (BSD)
	- 프린트 대기열의 상태를 출력. 프린터나 클래스 지정 안하면 기본 프린터로 출력함.
	- 옵션
		- -a : 모든 프린터의 인쇄 작업 정보 출력
		- -l : 프린터 대기열을 자세하게 출력
		- -P : 프린터 지정
		- +interval : 지정한 초마다 대기열이 비워질때까지 상태를 계속 출력
	- `lpq -P xp-610` : xp-610 프린터의 대기열 정보를 출력.
- lpstat (System V)
	- 옵션
		- -d : 기본 프린터 출력
		- -p : 설정된 모든 프린터, 출력 가능한 프린터 출력
		- -t : 모든 상태 정보 출력. 다른 옵션 다 때려박은거랑 동일
		- -a : 프린트 대기열이 요청 받을 수 있는 상태인지 출력
		- -c : 모든 클래스와 클래스에 속한 멤버 프린터가 뭔지 출력
		- -s : 요약 정보. summary
	- `lpstat -d` : 기본 프린터 정보 출력
	- `lpstat -s` : 프린터 요약 정보를 출력
	- `lpq -a xp-610` : xp-610이 요청 받을 수 있는지 확인

## 3. 사운드 카드
### 1. 리눅스 사운드의 개요
- 대표적으로 2가지가 있음.
	- 오픈 사운드 시스템 (OSS) : 유닉스 및 유닉스 호환 시스템
	- 어드밴스드 리눅스 사운드 아키텍쳐 (ALSA) : 리눅스에서만 지원.
- 리눅스 2.4까지는 OSS의 오디오 API를 썼지만 이후로는 ALSA를 씀.

### 2. 오픈 사운드 시스템 (OSS)
1. 정의
	- 안써
2. 목표
	- 사운드 하드웨어에 잘 알지 못해도 구현할 수 있는 기반이나 시스템 제공
3. OSS의 API
	- POSIX의 read, write, ioctl 같은 표준 시스템 호출을 기반으로 되어있는 API
	- 사운드 입출력의 기본 장치는 **/dev/dsp**.
	- ex) `cat /dev/random > /dev/dsp` : 해당 사운드 장치로 랜덤 값 (소음)을 쓰는 명령어
4. 라이선스
	- 초창기에는 자유 소프트웨어였음. 
	- 근데 나중에는 4Front라는 회사랑 계약함. 아 안궁금해

### 3. 고급 리눅스 사운드 아키텍쳐 (ALSA)
1. 정의
	- 안물
2. 목표
	- 안궁
3. 특징
	- 하드웨어 기반 미디 합성 기능 제공.
	- 다중 채널 하드웨어 믹싱 기능 지원
	- 전이중 입출력 제공
	- 멀티 프로세스, 쓰레드 안전 디바이스 드라이버 제공
4. 라이선스
	- GPL임

### 4. 사운드 명령어
- alsactl
	- ASLA 기반 사운드 카드 드라이버에 대한 고급 설정을 할 수 있는 명령어
	- 옵션
		- -d : 디버그 모드. 로그가 풍부함
		- -f : 환경설정 파일 지정. 기본 파일은 **/etc/asound.state**
	- 명령
		- store : 사운드 카드에 대한 현재 정보를 환경설정 파일에 저장.
		- restore : 환경설정 파일의 설정에 따라 선택된 사운드 카드에 반영.
		- init : 사운드 장치를 기본 기본 값으로 초기화. 디바이스가 알려진 장치가 아니면 에러코드 99를 반환한다.
	- `sudo aslactl init` : 초기화
	- `sudo aslactl restore -f asound.state` : 지정한 환경설정 파일로 설정을 되돌림
- aslamixer
	- 사운드 설정 변경, 볼륨을 조절하기 위한 그래픽 믹서 프로그램
		- 사용자 인터페이스는 커서 형태. (curses)
		- x 윈도우 필요 없음
	- 텍스트 기반인데도 그래픽 화면을 통해 사운드 카드 제어가 가능함.
	- 터미널용 옵션 있긴 한데 굳이?
- cdparanoia
	- 오디오 cd를 읽어서 재생 정보, 음악 파일을 추출함
	- 옵션
		- -w : 마이크로소프트 RIFF WAV 형식으로 출력
		- -a : 애플 AIFF-C 형식
		- -B : 모든 트랙의 음악을 Cdda2wav 스타일로 출력. 추출된 파일은 "track#. " 파일 형식을 가짐
		- -- 숫자 : 특정 트랙을 추출. 범위로도 설정이 가능.
	- 예시
		- `cdparanoia -B -- -1-2` : 1,2번 트랙 추출
		- `cdparanoia -B "1[:13.13]-1[1:13]"` : 1번 트랙의 0:13.13부터 1:13.00 사이를 추출함. `[분:초.밀리초]` 형식이고 빈칸이면 0을 뜻하는 듯.

## 4. 스캐너
### 1. SANE (스캐너 액세스 나우 이지)
1. 설명
	- 네이밍 센스 ㅅㅂㅋㅋㅋㅋㅋㅋㅋㅋ
	- 이미지 스캐너나 카메라 하드웨어에 대한 표준화 액세스를 제공하는 자유 사용 가능한 API임.
2. 아키텍쳐

3. 지원 OS
	- 이빠이 지원함
4. 오픈소스 라이선스
	- GPL임

### 2. XSANE (X 베이스드 인터페이스 포 더 SANE)
1. 설명
	- GTK+로 개발된 그래픽 기반 프로그램
	- 윈도우의 경우 로컬 스캐너를 지원하지 않는 것을 제외하면 다른 운영체제는 거의 다 지원함.
2. 기능
	- xsane은 독립 실행 / GIMP 이미지 조작 프로그램으로 실행 가능.
	- 다양한 형식의 이미지 저장 가능
	- 팩트나 프린터로 이미지 전송 가능
3. 설치
	- `sudo yum install xsane`
	- 실행은 그냥 `xsane` 입력

### 3. 스캐너 관련 명령어
- sane-find-scanner
	- SANE 백엔드를 지원하는 SANE USB, SCSI 스캐너를 검색하기 위한 명령어.
	- SCSI 스캐너를 찾기 위해 **/dev/sg0, /dev/scanner**와 같은 기본 SCSI 장치 파일을 검색
	- USB 스캐너를 찾기 위해서는 위 마지막 폴더 앞에 usb가 붙음. 뭐 대충 보면 알아요
	- 병렬 포트로 연결된 대부분의 스캐너는 찾지 못하지만 -p 옵션으로 검색될 가능성은 있다.
	- 옵션
		- -q : 별도 설명 없이 스캐너 장치만 출력
		- -v : 옵션을 한번 주면 장치 이름과 테스트 결과를 포함한 자세한 정보를 출력. 두 번 주면 SCSI INQUERY 정보와 USB 디바이스 디스크립터 정보도 보여준다.
		- -p : 병렬 포트를 스캔
	- 예시
		- `sane-find-scanner /dev/scanner` : /dev/scanner에 연결된 SCSI 스캐너만 검색.
- scanimage
	- 평면 스캐너나 카메라 같은 장치의 이미지 스캔을 제어함. 스캔된 이미지는 PNM이나 TIFF 형식. 
	- 옵션
		- -d : 장치 이름을 설정. 장치 이름은 `제조사:경로`
		- --format : 이미지 형식 지정. 기본은 pnm.
		- -L : 사용 가능한 스캐너 장치 목록 출력
	- 예시
		- `scanimage -x 100 -y 100 --format=tiff>image.tiff` : 스캔하고 이미지를 생성
- scanadf
	- 자동용지급지장치 (ADF : Automatic Document Feeder) 기능이 있는 스캐너에서 여러 장의 이미지를 스캔함. 장치가 SANE_STATUS_NO_DOCS 상태 코드를 반환할 때까지 스캔을 수행함.
	- 옵션
		- -d : 장치 파일명
		- -L : 목록 출력
		- -o : 스캔 결과물 파일 지정
	- 예시
		- `scanadf -d epson:/dev/sg0 --resolution 300 -o image_%04d` : 스캐너를 사용하여 300 dpi로 캡처, image_0000, image_0001 형식으로 이미지 파일 생성.
